### 1. 请简述Pandas中Series和DataFrame的区别，以及各自的适用场景。

#### 参考答案：

核心区别：维度不同、存储数据类型不同、用途不同。

1. Series：
- 一维数据结构，仅包含一列数据（值）和一列索引（index）；
- 数据类型可以是任意（int、str、float等），但同一Series中数据类型一致；
- 适用场景：存储单一维度的数据（如某一列指标、时间序列数据），常用于数据筛选、简单统计（如某字段的均值、中位数）。

2. DataFrame：
- 二维表格型数据结构，包含行索引（index）和列索引（columns），由多个Series组成（每一列都是一个Series）；
- 不同列可以有不同的数据类型（如一列int、一列str、一列datetime）；
- 适用场景：数据治理、数据中台核心数据载体，用于存储结构化数据（如业务数据表、清洗后的数据），支持多列数据的关联、筛选、聚合、清洗等操作，是数据处理的核心对象。

### 2. 请用Numpy实现两个二维数组的矩阵乘法，以及两个一维数组的点积，并说明两者的区别。

#### 参考答案：

1. 代码实现：

```python
import numpy as np

# 二维数组矩阵乘法（要求第一个数组的列数 = 第二个数组的行数）
arr1 = np.array([[1, 2], [3, 4]])  # 2行2列
arr2 = np.array([[5, 6], [7, 8]])  # 2行2列
matrix_mult = np.dot(arr1, arr2)  # 方法1：np.dot
# 或 matrix_mult = arr1 @ arr2  # 方法2：@运算符
print("二维矩阵乘法结果：\n", matrix_mult)

# 一维数组点积（对应元素相乘后求和，要求两个数组长度一致）
arr3 = np.array([1, 2, 3])
arr4 = np.array([4, 5, 6])
dot_product = np.dot(arr3, arr4)  # 等价于 arr3.dot(arr4)
print("一维数组点积结果：", dot_product)
```

2. 输出结果：

二维矩阵乘法结果：
 [[19 22]
 [43 50]]
一维数组点积结果： 32（1*4 + 2*5 + 3*6 = 4+10+18=32）

3. 核心区别：
- 矩阵乘法（二维）：遵循“前阵列数=后阵行数”的规则，结果是一个新的二维数组，维度为（前阵行数，后阵列数），本质是“行×列”的线性组合；
- 点积（一维）：本质是“元素对应相乘再求和”，结果是一个标量（单个数值），无维度要求（仅需长度一致），常用于计算向量的相似度、线性回归中的参数计算。

### 3. 简述Scikit-learn中train_test_split函数的作用，以及参数test_size、random_state的含义。

#### 参考答案：

1. 核心作用：将数据集（特征集X、标签集y）随机划分为训练集（用于模型训练）和测试集（用于模型评估），避免模型过拟合，验证模型的泛化能力，是机器学习建模的基础步骤。

2. 关键参数含义：
- test_size：指定测试集占总数据集的比例（取值0~1），例如test_size=0.2表示测试集占20%，训练集占80%；也可指定具体样本数量（如test_size=100，表示测试集取100个样本）。
- random_state：随机种子，用于固定随机划分的结果。当random_state取固定值（如0、42）时，每次运行train_test_split，划分的训练集和测试集完全一致，便于实验复现；若不指定，每次划分结果会随机变化。

### 1. 数据治理场景中，经常遇到“缺失值”“重复值”“异常值”，请用Pandas实现以下操作（基于任意模拟数据集）：

（1）查看数据集的缺失值比例；（2）删除重复行（保留第一次出现的行）；（3）用“中位数”填充数值型字段的缺失值，用“众数”填充分类型字段的缺失值；（4）用IQR方法识别并处理数值型字段的异常值（将异常值替换为该字段的上四分位/下四分位）。

#### 参考答案：

```python
import pandas as pd
import numpy as np

# 1. 模拟数据集（贴合数据治理实际，包含数值型、分类型字段，故意设置缺失值、重复值、异常值）
data = pd.DataFrame({
    "user_id": [1, 2, 2, 3, 4, 5, 6, 7, 8, 9],  # 重复值（user_id=2）
    "age": [25, 32, None, 40, 100, 35, None, 28, 36, 31],  # 缺失值、异常值（100）
    "gender": ["男", "女", "女", None, "男", "女", "男", None, "女", "男"],  # 缺失值
    "salary": [8000, 9500, 9500, None, 15000, 8500, 12000, 7800, 9000, 20000]  # 缺失值、异常值（20000）
})

# （1）查看各字段缺失值比例
missing_rate = data.isnull().sum() / len(data) * 100
print("各字段缺失值比例（%）：\n", missing_rate.round(2))

# （2）删除重复行（保留第一次出现的行，基于所有列判断重复）
data = data.drop_duplicates(keep="first")

# （3）填充缺失值：数值型用中位数，分类型用众数
# 数值型字段（age、salary）
numeric_cols = data.select_dtypes(include=[np.number]).columns
for col in numeric_cols:
    median_val = data[col].median()
    data[col].fillna(median_val, inplace=True)

# 分类型字段（gender）
category_cols = data.select_dtypes(include=["object"]).columns
for col in category_cols:
    mode_val = data[col].mode()[0]  # mode()返回Series，取第一个众数
    data[col].fillna(mode_val, inplace=True)

# （4）用IQR方法处理数值型字段异常值
def handle_outliers_iqr(df, col):
    # 计算四分位
    q1 = df[col].quantile(0.25)
    q3 = df[col].quantile(0.75)
    iqr = q3 - q1
    # 异常值阈值：小于q1-1.5*iqr 或 大于q3+1.5*iqr
    lower_bound = q1 - 1.5 * iqr
    upper_bound = q3 + 1.5 * iqr
    # 替换异常值：低于下界替换为q1，高于上界替换为q3
    df.loc[df[col] < lower_bound, col] = lower_bound
    df.loc[df[col] > upper_bound, col] = upper_bound
    return df

# 对所有数值型字段处理异常值
for col in numeric_cols:
    data = handle_outliers_iqr(data, col)

print("\n处理后的数据：\n", data)
```

核心说明：数据治理中，缺失值/异常值处理需结合业务场景（如本题用中位数/众数是通用方案，实际可能需结合业务逻辑填充）；IQR方法是稳健的异常值识别方法，适用于非正态分布的数值型数据，比“3σ原则”更不易受极端值影响。

### 2. 数据中台场景中，需对多表数据进行关联整合，请用Pandas实现以下操作：

现有两张表：表1（user_info）包含user_id（用户ID）、user_name（用户名）、register_time（注册时间）；表2（user_behavior）包含user_id（用户ID）、behavior_type（行为类型：浏览/下单/支付）、behavior_time（行为时间）。请实现：（1）将两张表通过user_id进行内连接，得到“用户信息+用户行为”的完整数据；（2）统计每个用户的行为总数，新增“behavior_count”字段；（3）筛选出“注册时间在2024-01-01之后，且有支付行为”的用户数据。

#### 参考答案：

```python
import pandas as pd
from datetime import datetime

# 1. 模拟两张表数据
user_info = pd.DataFrame({
    "user_id": [101, 102, 103, 104, 105],
    "user_name": ["张三", "李四", "王五", "赵六", "孙七"],
    "register_time": ["2023-12-25", "2024-01-05", "2024-02-10", "2023-11-30", "2024-03-01"]
})

user_behavior = pd.DataFrame({
    "user_id": [101, 101, 102, 103, 103, 104, 105, 105],
    "behavior_type": ["浏览", "下单", "支付", "浏览", "支付", "浏览", "下单", "支付"],
    "behavior_time": ["2024-01-02", "2024-01-03", "2024-01-06", "2024-02-11", "2024-02-15", "2024-01-01", "2024-03-02", "2024-03-03"]
})

# （1）内连接两张表（只保留两张表中都有user_id的记录）
merged_data = pd.merge(user_info, user_behavior, on="user_id", how="inner")

# （2）统计每个用户的行为总数，新增behavior_count字段
# 方法：按user_id分组，统计每组的行数（即行为数），再合并回原数据
behavior_count = merged_data.groupby("user_id").size().reset_index(name="behavior_count")
merged_data = pd.merge(merged_data, behavior_count, on="user_id", how="left")

# （3）筛选条件：注册时间>2024-01-01 且 有支付行为
# 第一步：转换注册时间为datetime类型
merged_data["register_time"] = pd.to_datetime(merged_data["register_time"])
# 第二步：筛选注册时间在2024-01-01之后的记录
filtered_data1 = merged_data[merged_data["register_time"] > datetime(2024, 1, 1)]
# 第三步：筛选有支付行为的用户（先获取有支付行为的user_id，再筛选）
pay_user_ids = merged_data[merged_data["behavior_type"] == "支付"]["user_id"].unique()
filtered_data = filtered_data1[filtered_data1["user_id"].isin(pay_user_ids)]

print("最终筛选结果：\n", filtered_data)
```

核心说明：数据中台的核心是“数据整合”，Pandas的merge方法是多表关联的核心工具，需熟练掌握内连接、左连接（如保留所有用户信息，即使无行为数据）的区别；分组统计（groupby）是数据聚合的基础，贴合数据中台的指标计算场景。

### 3. 数据治理中，特征工程是建模的基础，请用Scikit-learn实现以下特征工程操作：

现有一份用户特征数据（包含数值型特征、分类型特征），请实现：（1）对分类型特征进行独热编码（One-Hot Encoding）；（2）对数值型特征进行标准化（StandardScaler）；（3）使用SelectKBest筛选出与标签相关性最高的5个特征；（4）将处理后的特征和标签划分成训练集和测试集（测试集占20%，固定随机种子为42）。

#### 参考答案：

```python
import pandas as pd
import numpy as np
from sklearn.preprocessing import OneHotEncoder, StandardScaler
from sklearn.feature_selection import SelectKBest, f_classif
from sklearn.model_selection import train_test_split

# 1. 模拟特征数据和标签（贴合用户场景，包含数值型、分类型特征）
data = pd.DataFrame({
    "age": [25, 32, 40, 35, 28, 36, 45, 30],  # 数值型特征
    "salary": [8000, 9500, 15000, 8500, 12000, 9000, 18000, 7800],  # 数值型特征
    "gender": ["男", "女", "男", "女", "男", "女", "男", "女"],  # 分类型特征
    "city": ["北京", "上海", "广州", "北京", "上海", "广州", "北京", "上海"],  # 分类型特征
    "label": [0, 1, 1, 0, 1, 0, 1, 0]  # 标签（如用户是否留存、是否转化）
})

# 拆分特征（X）和标签（y）
X = data.drop("label", axis=1)
y = data["label"]

# 拆分数值型特征和分类型特征
numeric_cols = X.select_dtypes(include=[np.number]).columns
category_cols = X.select_dtypes(include=["object"]).columns
X_numeric = X[numeric_cols]
X_category = X[category_cols]

# （1）分类型特征：独热编码（One-Hot Encoding）
encoder = OneHotEncoder(sparse_output=False, drop="first")  # drop="first"避免多重共线性
X_category_encoded = encoder.fit_transform(X_category)
# 将编码后的结果转换为DataFrame，便于后续合并
X_category_df = pd.DataFrame(
    X_category_encoded,
    columns=encoder.get_feature_names_out(category_cols)
)

# （2）数值型特征：标准化（StandardScaler，均值为0，方差为1）
scaler = StandardScaler()
X_numeric_scaled = scaler.fit_transform(X_numeric)
X_numeric_df = pd.DataFrame(
    X_numeric_scaled,
    columns=numeric_cols
)

# 合并处理后的数值型和分类型特征，得到完整特征集
X_processed = pd.concat([X_numeric_df, X_category_df], axis=1)

# （3）筛选与标签相关性最高的5个特征（SelectKBest，基于F检验）
selector = SelectKBest(score_func=f_classif, k=5)
X_selected = selector.fit_transform(X_processed, y)
# 查看筛选后的特征名称
selected_feature_names = X_processed.columns[selector.get_support()]
print("筛选出的5个最优特征：", selected_feature_names.tolist())

# （4）划分训练集和测试集
X_train, X_test, y_train, y_test = train_test_split(
    X_selected, y, test_size=0.2, random_state=42, stratify=y  # stratify=y保证标签分布均匀
)

print("\n训练集特征形状：", X_train.shape)
print("测试集特征形状：", X_test.shape)
```

核心说明：数据治理中，特征工程的核心是“让特征更适合建模”——独热编码解决分类型特征无法直接建模的问题，标准化解决数值型特征量级差异导致的模型偏差；特征筛选（SelectKBest）可减少冗余特征，提升模型效率，贴合数据中台“数据提纯”的核心需求。

```python
import pandas as pd
import numpy as np

# 步骤1：读取原始订单数据（模拟数据，贴合实际业务）
order_data = pd.read_csv("order_data.csv")  # 实际场景中从业务数据库读取

# 步骤2：数据探查（查看数据基本信息、缺失值、异常值）
print("原始数据基本信息：")
print(order_data.info())
print("缺失值统计：\n", order_data.isnull().sum())
print("订单金额描述统计：\n", order_data["order_amount"].describe())

# 步骤3：数据清洗
# （1）处理重复订单（按order_id去重，保留最新的一条，基于create_time）
order_data["create_time"] = pd.to_datetime(order_data["create_time"], errors="coerce")  # 转换时间格式，异常值设为NaT
order_data = order_data.sort_values("create_time").drop_duplicates(subset="order_id", keep="last")

# （2）处理时间格式统一（将支付时间的字符串/时间戳统一转为datetime格式）
def unify_time_format(time_val):
    if pd.isna(time_val):
        return pd.NaT
    # 处理时间戳（假设为10位时间戳）
    if isinstance(time_val, (int, float)) and len(str(int(time_val))) == 10:
        return pd.to_datetime(time_val, unit="s")
    # 处理字符串格式（如"2024-01-01 10:00:00"）
    try:
        return pd.to_datetime(time_val)
    except:
        return pd.NaT

order_data["pay_time"] = order_data["pay_time"].apply(unify_time_format)

# （3）处理缺失值：订单金额用中位数填充，支付时间用“创建时间+2小时”填充（贴合业务逻辑）
order_data["order_amount"].fillna(order_data["order_amount"].median(), inplace=True)
order_data["pay_time"].fillna(order_data["create_time"] + pd.Timedelta(hours=2), inplace=True)

# （4）处理异常值：订单金额<0或>100000视为异常，替换为中位数
median_amount = order_data["order_amount"].median()
order_data.loc[(order_data["order_amount"] < 0) | (order_data["order_amount"] > 100000), "order_amount"] = median_amount

# 步骤4：生成标准化数据表（添加数据治理标识字段）
order_data["data_governance_time"] = pd.Timestamp.now()  # 数据治理时间
order_data["data_quality"] = "合格"  # 标记数据质量（可根据实际规则调整）

# 步骤5：保存治理后的数据（用于数据中台复用）
order_data.to_csv("standardized_order_data.csv", index=False)

print("\n数据治理完成！治理后数据形状：", order_data.shape)
print("治理后数据缺失值：\n", order_data.isnull().sum())
```
