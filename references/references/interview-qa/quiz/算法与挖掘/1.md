# 算法 & 挖掘模拟面试卷（5 题 + 标准答案）

适用岗位：数据挖掘 / 算法工程师 / 数据中台算法岗

## 1. 请简述特征工程的核心步骤，并说明在 XGBoost 中做特征工程与 PCA 的区别与适用场景

### **参考答案**

1）**特征工程核心步骤**：数据清洗→特征构造→特征选择→特征转换→特征监控。

2）**XGBoost 特征工程**：树模型对异常值、量纲不敏感，重点是**交叉特征、离散化、统计特征、缺失值处理、类别特征编码**，不需要强标准化，可输出特征重要性做筛选。

3）**PCA**：无监督降维，消除共线性、降低维度与计算量，但会**丢失可解释性**，适合高稀疏特征、图像 / 文本 Embedding 等场景。

4）**区别**：PCA 是降维变换，XGBoost 是在原特征空间做筛选与构造；PCA 不适合需要强可解释性的业务建模，XGBoost 更适合结构化数据建模。

## 2. 请对比常见聚类算法（K-Means / DBSCAN / 层次聚类）的优缺点与适用场景

### **参考答案**

- **K-Means**：快、可扩展强；需指定 K、对噪声 / 离群点敏感、只适合凸簇。适用于海量结构化数据用户分群。
- **DBSCAN**：不需指定 K、能识别任意形状簇、自动识别噪声；高维表现差、密度不均时效果差。适用于异常检测、地理点云聚类。
- **层次聚类**：无需 K、可生成树状谱系图；复杂度高、不适合大数据。适用于小样本、需要层级结构的场景。

## 3. 什么是 BM25 算法？它的核心公式思想是什么？相比 TF-IDF 优势在哪？

### **参考答案**

BM25 是基于概率检索模型的**相关性排序算法**，常用于搜索引擎 / 检索召回。

1）核心思想：综合**词频 TF、逆文档频率 IDF、文档长度归一化**进行打分，抑制过长文档的 “词频膨胀”。

2）相比 TF-IDF 优势：

- 引入**文档长度归一化**，避免长文档天然占优
- 词频满足**饱和机制**，高频词不会无限提升得分
- 更适合真实检索排序，在 RAG 检索召回中广泛替代 TF-IDF

## 4. 完整描述 RAG 技术体系流程，并说明检索阶段为什么常使用 “BM25 + 向量检索” 混合召回？

### **参考答案**

1）**RAG 标准流程**：文档切分→Embedding 向量化→向量库建索引→用户 Query 向量化→检索召回→重排序→输入 LLM 生成回答。

2）**混合召回原因**：

- **向量检索**擅长语义相似、泛化能力强，但对精确关键词匹配弱。
- **BM25**擅长关键词精确匹配、字面重合度高，可弥补向量语义漂移。
- 两者互补，提升召回覆盖率与相关性，减少漏召回与幻觉。

## 5. 分类建模中，XGBoost 为什么通常优于传统 LR？在实际业务中如何处理类别不平衡问题？

### **参考答案**

1）**XGBoost 优于 LR 原因**：

- 能自动拟合**非线性、交互特征**，LR 需要手动构造
- 内置正则，抗过拟合更强
- 对缺失值、异常值更鲁棒
- 支持特征重要性、缺失分裂、加权分位数等优化
    
    2）**类别不平衡处理**：
- 采样：过采样少数类、欠采样多数类、SMOTE
- 权重：class_weight /pos_weight
- 阈值移动：降低预测正类阈值
- 评估指标：用 F1、AUC、PR 曲线，不只用准确率
