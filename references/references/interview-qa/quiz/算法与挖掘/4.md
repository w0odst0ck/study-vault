## 1. 业务建模题

**题目**

公司要做一个**用户流失预警模型**，特征包括用户基本信息、行为日志、消费记录、客服反馈等。请你完整描述从数据到上线的全流程，并说明：

1）如何做特征工程；

2）样本不平衡如何处理；

3）模型选择与评估指标；

4）上线后如何监控与迭代。

### **参考答案**

1. **流程**
    
    需求定义→数据拉取与对齐→数据清洗→特征工程→样本构建→训练验证→评估→上线→A/B 测试→监控迭代。
    
2. **特征工程**
    

- 基础：用户人口属性、注册时长、最近登录时间。
- 行为：近 7/14/30 日访问频次、使用时长、页面跳转、功能渗透率。
- 消费：订单数、客单价、最近消费间隔、复购率。
- 文本：客服文本做情感得分、关键词（卡顿、闪退、贵、不好用）。
- 构造：变化率、环比、累计统计、缺失标记、时间差特征。

3. **样本不平衡**

- 训练时使用 `scale_pos_weight`；
- 少样本过采样 / SMOTE，多样本欠采样；
- 优先用 F1、Precision/Recall、AUC、PRC。

4. **模型与评估**

- 优先 XGBoost/LightGBM，兼顾效果与可解释性；
- 按时间划分训练 / 验证集，不随机打乱；
- 线上看精准率，保证触达用户确实高风险。

5. **监控与迭代**

- 监控特征分布漂移（PSI）、模型效果衰减；
- 按月重训，新增行为特征，持续 Badcase 分析。

## 2. 聚类场景题

**题目**

某电商要做**用户分群**，用于精细化运营。现有用户行为 Embedding、商品偏好向量、用户画像特征。

1）为什么直接 K-Means 效果差？

2）你会选择什么算法，为什么？

3）如何评价聚类效果，如何落地应用？

### **参考答案**

1. **K-Means 差的原因**

- 高维向量欧氏距离失效；
- 用户簇形状不规则、密度不均；
- 存在大量噪声用户。

2. **方案选择**

- 先用 UMAP 降维；
- 再用 **HDBSCAN**，无需指定 K、自动识别噪声、适合任意形状簇；
- 运营需要可解释性时，再用 K-Means 做二次分群。

3. **评估与落地**

- 内部指标：轮廓系数、Calinski-Harabasz；
- 业务指标：簇内同质性、转化率差异、人群稳定性；
- 落地：高价值用户专属权益、沉睡用户召回、低价值用户低成本运营。

## 3. RAG 系统设计题

**题目**

公司要做一个**内部知识库问答机器人**，文档为 PDF/Word/ 表格，含大量专业术语、表格数据、制度条款，容易出现幻觉与检索不准。

请设计一套完整可上线的 RAG 架构。

### **参考答案**

1. **文档处理**

- 解析：PDF/Word 提取文本 + 表格结构化；
- 分块：固定长度 + 滑动窗口，保留标题层级；
- 元数据：文件名、章节、更新时间。

2. **召回层**

- 混合召回：BM25（关键词 / 条款 / 编号）+ 向量检索（语义）；
- 各取 top50 后加权融合。

3. **精排层**

- CrossEncoder 重排；
- 规则过滤：包含核心术语、时间、条款号优先。

4. **生成层**

- Prompt 强制：只依据参考资料，标注来源；
- 表格结构化输出，避免转述错误；
- 加入引用校验与事实校验。

5. **工程与优化**

- 向量库：FAISS / Milvus；
- 缓存高频问题；
- 定期增量更新与重建索引。

## 4. 特征与模型原理题

**题目**

同样是树模型，XGBoost、LightGBM、CatBoost 核心差异是什么？在实际业务中你如何选择？

### **参考答案**

- **XGBoost**：二阶泰勒、正则完备、按层生长，精度稳但速度偏慢。
- **LightGBM**：按叶生长、直方图优化、GOSS/EFB，速度极快，工业界最常用。
- **CatBoost**：自动处理类别特征、解决预测偏移，类别特征多、低延迟场景更优。

**选择**

- 类别特征多、强时序数据 → CatBoost；
- 大数据量、追求速度 → LightGBM；
- 竞赛、追求极致精度 → XGBoost。

## 5. 检索算法综合题

**题目**

在 RAG 中，向量检索语义好但关键词弱，BM25 关键词准但语义弱。

请设计一套**混合召回 + 重排序**的完整策略，并说明如何做线上效果对比。

### **参考答案**

1. **召回策略**

- BM25：抓术语、编号、专有名词；
- 向量检索：抓意图、同义、泛化表述；
- 多路召回后去重，简单加权融合。

2. **重排序**

- 先用轻量模型快速粗排；
- 再用 CrossEncoder 精排 top10；
- 规则增强：命中实体加分、来源权威加分。

3. **线上对比**

- 指标：召回准确率、MRR、HitRate、LLM 回答准确率；
- 实验：A/B 测试，单向量 vs 单 BM25 vs 混合召回；
- Badcase 分析：漏召回、错召回、幻觉来源。
