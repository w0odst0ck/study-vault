动态规划（Dynamic Programming，简称 DP）是一种通过把原问题分解为相对简单的子问题，并保存子问题的解来避免重复计算，从而解决复杂问题的方法。以下是 C++ 中动态规划框架的一般操作步骤：

### 1. 问题分析

- **确定问题类型**：明确问题是否适合用动态规划解决，通常问题具有最优子结构性质（问题的最优解可以由子问题的最优解推导得出）和重叠子问题性质（子问题会被多次求解）。
- **定义状态**：这是动态规划的关键，需要找出能描述问题子问题的变量。这些变量的不同取值组合代表不同的状态，状态应具有无后效性，即某阶段状态一旦确定，就不受后续决策影响。例如，在背包问题中，状态可以定义为背包容量和物品数量。

### 2. 状态转移方程推导

- **分析子问题关系**：研究不同状态之间是如何转移的，即如何从已知的子问题解推导出更大规模子问题的解。例如，在计算斐波那契数列时，第 `n` 项的值依赖于第 `n - 1` 项和第 `n - 2` 项。
- **得出转移方程**：用数学公式表达状态之间的转移关系。以背包问题为例，如果 `dp[i][j]` 表示考虑前 `i` 个物品，背包容量为 `j` 时能获得的最大价值，那么状态转移方程可能是 `dp[i][j] = max(dp[i - 1][j], dp[i - 1][j - weight[i]] + value[i])`，表示不放入第 `i` 个物品和放入第 `i` 个物品两种情况下的最大价值。

### 3. 初始化

- **确定初始状态**：为状态转移提供起点，通常是问题的边界条件。例如，在斐波那契数列中，`dp[0] = 0`，`dp[1] = 1`；在背包问题中，当背包容量为 `0` 或者物品数量为 `0` 时，能获得的价值为 `0`，即 `dp[0][j] = 0`（`j` 为任意背包容量），`dp[i][0] = 0`（`i` 为任意物品数量）。

### 4. 状态计算与存储

- **选择存储结构**：根据定义的状态，选择合适的数据结构来存储状态值。常见的有一维数组、二维数组等。如果状态维度较高，也可能使用三维数组甚至更复杂的数据结构。例如，二维状态 `dp[i][j]` 可以用二维数组 `vector<vector<int>> dp(n, vector<int>(m))` 来存储，其中 `n` 和 `m` 分别是两个状态变量的取值范围。
- **按序计算状态**：根据状态转移方程，按照一定顺序填充存储状态的结构。通常从较小规模的子问题开始，逐步计算到较大规模的子问题。例如，在背包问题中，一般外层循环遍历物品数量 `i`，内层循环遍历背包容量 `j`，即 `for (int i = 1; i <= numItems; ++i) { for (int j = 1; j <= bagCapacity; ++j) { /* 根据状态转移方程计算 dp[i][j] */ } }`。

### 5. 结果获取

- **确定目标状态**：明确最终需要的解对应哪个状态值。例如，在背包问题中，最终目标可能是 `dp[numItems][bagCapacity]`，即考虑所有物品且背包容量为最大容量时能获得的最大价值。
- **提取结果**：从存储状态值的数据结构中获取目标状态的值，即为问题的解。

### 6. 空间优化（可选）

- **分析可优化空间**：有些动态规划问题中，某些状态在计算完后续状态后不再使用，可以通过优化存储方式减少空间占用。例如，在背包问题中，每个状态 `dp[i][j]` 只依赖于 `dp[i - 1][j]` 和 `dp[i - 1][j - weight[i]]`，因此可以将二维数组优化为一维数组，通过滚动数组实现空间复杂度从 `O(n * m)` 降低到 `O(m)`。
- **实现空间优化**：根据分析结果，调整存储结构和状态转移计算过程。例如，将二维数组 `dp[i][j]` 改为一维数组 `dp[j]` 后，状态转移方程变为 `dp[j] = max(dp[j], dp[j - weight[i]] + value[i])`，同时内层循环需要逆序遍历背包容量 `j`，即 `for (int i = 1; i <= numItems; ++i) { for (int j = bagCapacity; j >= weight[i]; --j) { /* 计算 dp[j] */ } }`，以避免覆盖未使用的状态值。
