在 C++ 中，搜索操作是在数据集合中查找特定元素的过程，常见的搜索算法框架包括线性搜索、二分搜索以及基于树和图结构的搜索，如深度优先搜索（DFS）和广度优先搜索（BFS）。以下是这些搜索框架的详细操作：

### 线性搜索

1. **基本思想**：从数据集合的开头开始，逐个检查元素，直到找到目标元素或遍历完整个集合。
2. **操作步骤**：
    
    - 从集合的第一个元素开始。
    - 将当前元素与目标元素进行比较。
    - 如果当前元素等于目标元素，则返回当前元素的位置。
    - 如果当前元素不等于目标元素，则移动到下一个元素，重复上述比较过程。
    - 如果遍历完整个集合都没有找到目标元素，则返回表示未找到的特殊值（例如 -1）。
    

### 二分搜索

1. **适用条件**：数据集合必须是有序的。
2. **基本思想**：通过不断将搜索区间缩小一半，快速定位目标元素。
3. **操作步骤**：
    
    - 初始化搜索区间，设置左边界 `left` 为集合的起始位置，右边界 `right` 为集合的末尾位置。
    - 当 `left` 小于等于 `right` 时，执行以下循环：
        
        - 计算中间位置 `mid = left + (right - left) / 2`。
        - 将中间位置的元素与目标元素进行比较：
            
            - 如果中间元素等于目标元素，则返回中间位置 `mid`。
            - 如果中间元素大于目标元素，则将右边界 `right` 更新为 `mid - 1`，缩小搜索区间到左半部分。
            - 如果中间元素小于目标元素，则将左边界 `left` 更新为 `mid + 1`，缩小搜索区间到右半部分。
            
        
    - 如果循环结束仍未找到目标元素，则返回表示未找到的特殊值（例如 -1）。
    

### 深度优先搜索（DFS）

1. **适用结构**：常用于树和图结构。
2. **基本思想**：从起始节点开始，尽可能深地探索分支，直到无法继续或找到目标节点，然后回溯到上一个节点，继续探索其他分支。
3. **操作步骤**：
    
    - **递归实现**：
        
        - 定义一个辅助函数，接受当前节点作为参数。
        - 在辅助函数中，首先检查当前节点是否为目标节点：
            
            - 如果是目标节点，则进行相应处理（例如返回当前节点或标记已找到）。
            - 如果不是目标节点，则标记当前节点为已访问。
            - 对当前节点的所有未访问邻接节点递归调用辅助函数。
            
        - 从起始节点调用辅助函数开始搜索。
        
    - **非递归实现（使用栈）**：
        
        - 创建一个栈，将起始节点压入栈中。
        - 创建一个集合用于记录已访问的节点。
        - 当栈不为空时，执行以下操作：
            
            - 从栈顶弹出一个节点。
            - 如果该节点是目标节点，则进行相应处理。
            - 如果该节点未被访问过：
                
                - 标记该节点为已访问。
                - 将该节点的所有未访问邻接节点压入栈中（顺序根据具体需求，例如从左到右或从右到左）。
                
            
        
    

### 广度优先搜索（BFS）

1. **适用结构**：同样常用于树和图结构，尤其适用于寻找最短路径问题。
2. **基本思想**：从起始节点开始，逐层探索节点，先访问距离起始节点近的节点，再逐渐扩展到距离远的节点。
3. **操作步骤**：
    
    - 创建一个队列，将起始节点放入队列中。
    - 创建一个集合用于记录已访问的节点。
    - 当队列不为空时，执行以下操作：
        
        - 从队列头部取出一个节点。
        - 如果该节点是目标节点，则进行相应处理（例如返回当前节点或标记已找到）。
        - 如果该节点未被访问过：
            
            - 标记该节点为已访问。
            - 将该节点的所有未访问邻接节点加入队列尾部。
            
        
    

### 基于哈希表的搜索

1. **适用场景**：当需要快速判断元素是否存在或查找元素的关联值时，可使用哈希表。
2. **基本思想**：通过哈希函数将元素映射到哈希表的特定位置，从而实现快速查找。
3. **操作步骤**：
    
    - **插入操作**：
        
        - 计算元素的哈希值，通过哈希函数将元素映射到哈希表的一个桶（bucket）。
        - 如果桶为空，则直接将元素插入该桶。
        - 如果桶不为空，处理哈希冲突（例如使用链地址法，将冲突的元素以链表形式存储在桶中）。
        
    - **查找操作**：
        
        - 计算目标元素的哈希值，定位到对应的桶。
        - 在桶中查找目标元素（如果使用链地址法，需要遍历链表）。
        - 如果找到目标元素，则返回相关信息（例如元素本身或其关联值）；否则返回表示未找到的特殊值。
        
    

这些搜索框架在不同场景下各有优劣，线性搜索适用于无序集合，但时间复杂度较高；二分搜索适用于有序集合，效率较高；DFS 和 BFS 适用于树和图结构，可解决路径查找等问题；哈希表搜索则在快速查找元素存在性和关联值方面表现出色。在实际编程中，应根据具体问题的特点选择合适的搜索框架。
