在 Python 中，迭代器是遵循迭代器协议的对象，它包含两个关键方法：`__iter__()` 和 `__next__()`。以下是 Python 迭代器框架涉及的所有操作：

### 1. 创建迭代器

- **可迭代对象生成迭代器**：任何实现了 `__iter__()` 方法的对象都是可迭代对象。通过对可迭代对象调用 `iter()` 函数，会返回一个迭代器。例如，列表、元组、字符串、字典等容器类型都是可迭代对象，可通过 `iter()` 函数获取对应的迭代器。
- **自定义迭代器类**：定义一个类，在类中实现 `__iter__()` 和 `__next__()` 方法。`__iter__()` 方法应返回迭代器对象本身（即 `self`），`__next__()` 方法用于返回下一个值，当没有更多值可返回时，应引发 `StopIteration` 异常。

### 2. 迭代器的核心方法

- **`__iter__()`**：返回迭代器对象本身。此方法使得迭代器对象可以在需要可迭代对象的环境中使用，例如在 `for` 循环中。
- **`__next__()`**：返回迭代器的下一个元素。每次调用 `__next__()` 时，迭代器应更新其内部状态，指向下一个元素。当到达迭代器末尾，没有更多元素可返回时，必须引发 `StopIteration` 异常，以通知调用者迭代结束。

### 3. 使用迭代器

- **在 `for` 循环中使用**：`for` 循环会自动调用迭代器的 `__iter__()` 方法获取迭代器，然后在每次循环迭代中调用 `__next__()` 方法获取下一个元素，直到捕获到 `StopIteration` 异常时停止循环。
- **手动使用 `next()` 函数**：可以手动调用内置的 `next()` 函数，并传入迭代器对象来逐个获取元素。例如 `value = next(iterator)`，其中 `iterator` 是迭代器对象。与迭代器的 `__next__()` 方法作用相同，但更方便在需要显式获取下一个元素的场景中使用。

### 4. 其他与迭代器相关的操作

- **解包迭代器**：可以使用解包操作将迭代器中的元素分配给多个变量。例如，对于包含三个元素的迭代器，可以使用 `a, b, c = iterator` 将迭代器的前三个元素分别赋值给 `a`、`b` 和 `c`。解包的变量数量必须与迭代器中的元素数量匹配（在 Python 3.5+ 中，也支持使用 `*` 进行部分解包）。
- **与其他函数结合使用**：许多 Python 内置函数和标准库函数都支持迭代器作为输入。例如，`list()`、`tuple()`、`set()` 等函数可以将迭代器转换为相应的容器类型；`sum()`、`min()`、`max()` 等函数可以对迭代器中的元素进行计算；`map()`、`filter()` 等函数可以对迭代器中的元素进行映射和过滤操作。

### 5. 生成器（特殊的迭代器）

- **生成器函数**：定义一个包含 `yield` 关键字的函数，该函数会返回一个生成器对象，生成器是一种特殊的迭代器。当生成器函数被调用时，它不会立即执行函数体，而是返回一个生成器对象。每次调用生成器的 `__next__()` 方法（或使用 `next()` 函数）时，生成器函数执行到 `yield` 语句处暂停，返回 `yield` 后面的值，并且保存函数的当前状态。下次调用 `__next__()` 时，从暂停的位置继续执行，直到遇到下一个 `yield` 或函数结束。
- **生成器表达式**：类似于列表推导式，但使用圆括号而不是方括号。例如，`(x * 2 for x in range(5))` 创建一个生成器对象，该生成器按需生成值，而不是像列表推导式那样立即生成所有值并占用内存。生成器表达式也是一种便捷的创建生成器的方式。
