## 一、口述基础题（5 题，每题 6 分，共 30 分）

### 1. Pandas 中 `loc` 和 `iloc` 的区别？数据治理中为什么推荐用 `loc`？

**参考答案**

- `loc`：按 **标签（行名 / 列名** 索引，支持条件筛选、标签切片（左闭右闭）。
- `iloc`：按 **位置下标（数字** 索引，左闭右开。
- 治理场景推荐 `loc`：避免索引重置后位置错乱、代码可读性强、支持条件赋值（如异常值替换），减少 `SettingWithCopyWarning`。

### 2. 数据治理中，处理缺失值有哪些常用策略？分别适用什么场景？

**参考答案**

- **删除**：缺失率 > 50%、无业务意义、样本充足时用（`dropna`）。
- **统计值填充**：数值型用**中位数**（抗异常）、分类型用**众数**；适合数据量中等、分布不规则。
- **业务填充**：时间缺失用「创建时间 + 固定间隔」、金额用「同类均值」；贴合业务最稳健。
- **模型填充**：KNN、随机森林补全；适合高价值数据、缺失率低但特征多。

### 3. Numpy 数组和 Python 列表的核心区别？数据中台为什么优先用 Numpy？

**参考答案**

- **存储**：ndarray 同质（同类型）连续存储；list 异质、分散指针。
- **性能**：Numpy 向量化运算（C 底层），比 list 循环快 10–100 倍。
- **功能**：支持矩阵运算、广播、统计函数；适合数值计算、特征工程、指标批量计算。
- 中台场景：指标加工、特征标准化、异常值检测等批量运算必须用 Numpy 提速。

### 4. Scikit-learn 中 `OneHotEncoder` 和 Pandas `get_dummies` 的区别？生产环境选哪个？

**参考答案**

- `get_dummies`：简单快速、直接返回 DataFrame；**无状态**，新数据编码不一致。
- `OneHotEncoder`：**有状态**（`fit` 保存类别）、支持 `drop='first'` 防共线性、可嵌入 Pipeline。
- 生产 / 中台选 `OneHotEncoder`：保证训练 / 预测编码一致、支持流式数据、符合工程规范。

### 5. 数据中台处理千万级大表时，Pandas 内存不够怎么办？

**参考答案**

- **分块读取**：`read_csv(chunksize=100000)` 逐块处理。
- **精简类型**：`astype('category')`、`int32/float32` 替代默认 64 位。
- **按需加载**：`usecols` 只读需要的列。
- **并行工具**：Modin、Dask 替代 Pandas；超大数据用 PySpark。
- **避免复制**：`inplace=True`、`df.loc` 原地修改。

## 二、Numpy 手写代码题（1 题，15 分）

**要求**：手写代码实现（不能查文档）

生成 (100, 4) 的标准正态分布数据，完成：

1. 按列求均值、标准差
2. 把每列**标准化**（均值 0、方差 1）
3. 找出所有 > 1.96σ（95% 置信）的异常值，替换为该列**中位数**
4. 计算处理后矩阵的**秩**（判断是否线性相关）

### **参考答案**

```python
import numpy as np

np.random.seed(66)
data = np.random.randn(100, 4)

# 1 列均值、标准差
col_mean = data.mean(axis=0)
col_std = data.std(axis=0)

# 2 列标准化
data_std = (data - col_mean) / col_std

# 3 异常值替换（1.96σ）
for i in range(4):
    col = data_std[:, i]
    median = np.median(col)
    # 异常掩码
    mask = np.abs(col) > 1.96
    data_std[mask, i] = median

# 4 矩阵秩
rank = np.linalg.matrix_rank(data_std)
print("矩阵秩：", rank)
```

## 三、Pandas 数据治理综合题（1 题，20 分）

**场景**：用户行为表（数据质量差），需完成**企业级治理**

字段：user_id, phone, register_time, pay_amount, behavior, dt

问题：缺失、重复、手机号脏数据、金额异常、时间格式混乱

**要求**：手写代码实现

1. 按 `dt` 分日期、`user_id` 去重（保留最新 `register_time`）
2. 手机号清洗：只留数字、长度≠11 标记为 `invalid`
3. 缺失处理：数值用中位数、分类用众数、时间用同 `dt` 均值
4. 金额异常：<0 或>5 倍 IQR 替换为该 `dt` 中位数
5. 新增字段：`data_quality`（优秀 / 合格 / 不合格）、`clean_time`

### **模拟数据 + 参考答案**

```python
import pandas as pd
import numpy as np

# 模拟脏数据
df = pd.DataFrame({
    "user_id": [1,1,2,3,4,4,5],
    "phone": ["13812345678","13812345678","abc123","1398765432","13577889900",None,"136112233445"],
    "register_time": ["2024-01-01","2024-01-02","2024/01/03",None,"2024-01-05","2024-01-06","2024-01-07"],
    "pay_amount": [100,100,-50,5000,800,None,99999],
    "behavior": ["click","click","pay","click",None,"pay","pay"],
    "dt": ["2024-01-01"]*7
})

# 1 去重（按dt+user_id，保留最新注册时间）
df["register_time"] = pd.to_datetime(df["register_time"], errors="coerce")
df = df.sort_values("register_time").drop_duplicates(["dt","user_id"], keep="last")

# 2 手机号清洗
df["phone"] = df["phone"].astype(str).str.replace(r"\D","", regex=True)
df["phone"] = df["phone"].where(df["phone"].str.len()==11, "invalid")

# 3 缺失值处理
# 数值：中位数
df["pay_amount"] = df["pay_amount"].fillna(df["pay_amount"].median())
# 分类：众数
df["behavior"] = df["behavior"].fillna(df["behavior"].mode()[0])
# 时间：同dt均值
dt_median_t = df[df["dt"]=="2024-01-01"]["register_time"].median()
df["register_time"] = df["register_time"].fillna(dt_median_t)

# 4 金额异常处理（IQR）
def clean_amount(s):
    q1,q3 = s.quantile([0.25,0.75])
    iqr = q3-q1
    lower, upper = q1-1.5*iqr, q3+5*iqr  # 宽松上界
    return s.clip(lower, upper)

df["pay_amount"] = clean_amount(df["pay_amount"])

# 5 质量评分与标记
df["null_cnt"] = df.isnull().sum(axis=1)
df["data_quality"] = df["null_cnt"].apply(
    lambda x: "优秀" if x==0 else "合格" if x<=1 else "不合格"
)
df["clean_time"] = pd.Timestamp.now()

print(df)
```

## 四、数据中台指标与特征工程（1 题，18 分）

**场景**：用户表 + 订单表，需加工**用户宽表**用于建模

表 1：user（user_id, age, city, channel, register_dt）

表 2：order（order_id, user_id, pay_amt, pay_dt, order_status）

**要求**：

1. 左连接保留全用户，计算用户指标：订单数、总金额、首末次支付、支付天数
2. 特征工程：
    
    - 城市 / 渠道：OneHot（drop first）
    - 年龄 / 金额：标准化
    - 时间差：`register_to_first_pay`（天）
    
3. 特征筛选：方差过滤 + 相关系数过滤（剔除高相关）
4. 划分训练 / 测试集（分层抽样）

### **参考答案**

```python
import pandas as pd
import numpy as np
from sklearn.preprocessing import OneHotEncoder, StandardScaler
from sklearn.feature_selection import VarianceThreshold
from sklearn.model_selection import train_test_split

# 模拟数据
user = pd.DataFrame({
    "user_id":[1,2,3,4,5],
    "age":[22,35,28,40,33],
    "city":["bj","sh","bj","gz","sh"],
    "channel":["ad","natural","ad","natural","ad"],
    "register_dt":pd.to_datetime(["2024-01-01","2024-01-02","2024-01-03","2024-01-04","2024-01-05"])
})
order = pd.DataFrame({
    "user_id":[1,1,2,3,3,5],
    "pay_amt":[100,200,150,50,300,250],
    "pay_dt":pd.to_datetime(["2024-01-02","2024-01-03","2024-01-05","2024-01-06","2024-01-07","2024-01-08"]),
    "order_status":[1,1,1,1,1,1]
})

# 1 指标加工
order_agg = order.groupby("user_id").agg(
    order_cnt=("order_id","count"),
    total_amt=("pay_amt","sum"),
    first_pay=("pay_dt","min"),
    last_pay=("pay_dt","max"),
    pay_days=("pay_dt","nunique")
).reset_index()
df = pd.merge(user, order_agg, on="user_id", how="left")
# 缺失填充（未支付用户）
df[["order_cnt","total_amt","pay_days"]] = df[["order_cnt","total_amt","pay_days"]].fillna(0)

# 时间差特征
df["register_to_first_pay"] = (df["first_pay"] - df["register_dt"]).dt.days.fillna(-1)

# 2 特征工程
cat_cols = ["city","channel"]
num_cols = ["age","total_amt","order_cnt","register_to_first_pay"]

# 独热编码
ohe = OneHotEncoder(drop="first", sparse_output=False)
cat_feat = ohe.fit_transform(df[cat_cols])
cat_df = pd.DataFrame(cat_feat, columns=ohe.get_feature_names_out())

# 标准化
scaler = StandardScaler()
num_feat = scaler.fit_transform(df[num_cols])
num_df = pd.DataFrame(num_feat, columns=num_cols)

X = pd.concat([num_df, cat_df], axis=1)
y = (df["order_cnt"]>0).astype(int)  # 标签：是否支付

# 3 特征筛选
# 方差过滤
var_sel = VarianceThreshold(0.01)
X_var = var_sel.fit_transform(X)
# 相关系数过滤（>0.7剔除）
X_corr = pd.DataFrame(X_var).corr().abs()
upper = X_corr.where(np.triu(np.ones(X_corr.shape), k=1).astype(bool))
to_drop = [col for col in upper.columns if any(upper[col]>0.7)]
X_selected = pd.DataFrame(X_var).drop(to_drop, axis=1)

# 4 分层划分
X_train,X_test,y_train,y_test = train_test_split(
    X_selected,y,test_size=0.2,random_state=42,stratify=y
)
print("训练集形状:", X_train.shape)
```

**问题**：数据中台上线后，下游报表频繁报错：指标不准、缺失率高、重复数据。你作为数据治理工程师，如何用 Python 搭建**常态化数据质量监控体系**？请讲全流程 + 核心代码思路。

## **参考答案（面试高分口述）**

### 1. 监控体系设计（全链路）

- **规则层**：主键唯一、非空校验、值域范围、逻辑一致性、格式规范。
- **计算层**：Python+Pandas 定时执行（Airflow 调度）。
- **报告层**：生成质量报告、异常明细、趋势看板。
- **处置层**：自动拦截不合格数据、告警、回流清洗。

### 2. 核心代码框架（可直接落地）

```python
# 质量规则配置
quality_rules = {
    "user_id": {"not_null": True, "unique": True},
    "pay_amount": {"min":0, "max":1e6},
    "phone": {"regex":r"^1\d{10}$"},
    "pay_dt": {"gt":"register_dt"}
}

# 质量检查函数
def data_quality_check(df, rules):
    report = []
    for col, rule in rules.items():
        # 非空检查
        if rule.get("not_null"):
            null_pct = df[col].isnull().mean()
            report.append({"col":col,"rule":"not_null","pass":null_pct==0})
        # 唯一检查
        if rule.get("unique"):
            dup_pct = df.duplicated(col).mean()
            report.append({"col":col,"rule":"unique","pass":dup_pct==0})
        # 值域检查
        if "min" in rule:
            invalid = (df[col]<rule["min"]).mean()
            report.append({"col":col,"rule":"min","pass":invalid==0})
    return pd.DataFrame(report)

# 执行检查
quality_report = data_quality_check(df, quality_rules)
# 异常数据导出
bad_data = df[df[["user_id","pay_amount"]].isnull().any(axis=1)]
bad_data.to_csv("quality_error.csv", index=False)
```

### 3. 效果

- 实时监控：每日自动运行、异常告警。
- 数据准确率：从 75% → 99.5%。
- 下游稳定：报表零报错、支撑数据产品可靠运行。

## 六、加分题（5 分）

**问题**：Pandas 中 `groupby` 后为什么推荐用 `agg()` 而非多个 `apply`？

### **参考答案**

- `agg()`：向量化、C 底层执行，速度比 `apply`（Python 循环）快 5–10 倍。
- 支持多函数、多列、自定义命名（`order_cnt=('id','count')`）。
- 内存更优：一次性聚合、减少中间对象。
- 中台指标加工必须用 `agg()`，保证大表性能。
