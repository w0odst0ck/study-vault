# 一、基础口述题（5 题 × 4 分 = 20 分）

1. Pandas 中 `loc` 和 `iloc` 区别？为什么数据治理优先用 `loc`？
2. 处理缺失值常用哪些方法？分别适合什么场景？
3. 为什么生产环境不用 `pd.get_dummies`，而用 `OneHotEncoder`？
4. Pandas 处理大表内存不足，有哪几种优化方式？
5. 什么是数据质量中的**逻辑一致性**？举例说明。

---

## 参考答案

1. `loc` 按标签索引，`iloc` 按位置索引。`loc` 更安全、可读性强，避免索引错乱与赋值警告。
2. 删除（缺失极高）、中位数 / 众数填充（通用）、业务填充（最可靠）、模型填充（高价值数据）。
3. `get_dummies` 无状态，新数据类别会导致特征维度不匹配；`OneHotEncoder` 可保存规则，保证线上线下一致。
4. 指定 dtype、只加载需要列、分块读取、用 category 类型、避免全表复制、使用 Dask/Spark。
5. 数据之间业务逻辑合法，如：支付时间 > 注册时间、订单金额 ≥ 0、年龄在合理区间。

# 二、NumPy 手写题（1 题 × 12 分 = 12 分）

生成 shape=(100, 5) 标准正态数组，完成：

1. 按列求均值、标准差
2. 每列标准化
3. 绝对值大于 2 的值替换为该列中位数
4. 输出处理后数组的均值与形状

```python
import numpy as np
np.random.seed(42)

# 1
data = np.random.randn(100, 5)
mean = data.mean(axis=0)
std = data.std(axis=0)

# 2
data_norm = (data - mean) / std

# 3
for i in range(5):
    col = data_norm[:, i]
    mask = np.abs(col) > 2
    data_norm[mask, i] = np.median(col)

# 4
print(data_norm.shape)
print(data_norm.mean())
```

# 三、Pandas 数据清洗治理题（1 题 × 24 分 = 24 分）

给定用户数据表，字段：

user_id, phone, age, register_time, pay_amount, city

要求手写代码实现：

1. 缺失率 > 30% 的列直接删除
2. 按 user_id 去重，保留最新注册时间
3. 手机号只保留数字，长度≠11 标记为 'invalid'
4. age <16 或> 70 替换为中位数
5. pay_amount < 0 置为 0，缺失用 0 填充
6. 新增 `data_quality`：无缺失为 'good'，否则 'bad'

## 模拟数据 + 答案

```python
import pandas as pd
import numpy as np

df = pd.DataFrame({
    'user_id': [1,1,2,3,4,5,5],
    'phone': ['13800138000','13800138000','1234',None,'13999999999','abc123','136111122223'],
    'age': [18,18,80,None,30,10,35],
    'register_time': pd.date_range('2024-01-01', periods=7),
    'pay_amount': [100,100,-50,None,800,300,500],
    'city': ['bj','bj','sh',None,'gz','sz','sz']
})

# 1
missing = df.isnull().sum() / len(df)
df = df.drop(columns=missing[missing>0.3].index, errors='ignore')

# 2
df = df.sort_values('register_time').drop_duplicates('user_id', keep='last')

# 3
df['phone'] = df['phone'].astype(str).str.replace(r'\D', '', regex=True)
df['phone'] = df['phone'].where(df['phone'].str.len()==11, 'invalid')

# 4
med_age = df['age'].median()
df['age'] = df['age'].clip(16, 70).fillna(med_age)

# 5
df['pay_amount'] = df['pay_amount'].fillna(0)
df.loc[df['pay_amount'] < 0, 'pay_amount'] = 0

# 6
df['data_quality'] = df.isnull().any(axis=1).map({False: 'good', True: 'bad'})
```

两张表：

- user：user_id, channel, register_dt

要求：

1. 左连接保留所有用户
2. 统计每个用户：订单数、总支付金额、首次支付时间
3. 筛选 2024 年注册、渠道为 'ad' 的付费用户
4. 按渠道计算付费转化率（付费用户数 / 总用户数）

```python
import pandas as pd

user = pd.DataFrame({
    'user_id': [1,2,3,4,5],
    'channel': ['ad','natural','ad','natural','ad'],
    'register_dt': pd.to_datetime([
        '2024-01-01','2023-12-01','2024-02-01','2024-03-01','2023-11-01'
    ])
})

order = pd.DataFrame({
    'order_id': [101,102,103,104],
    'user_id': [1,1,3,3],
    'pay_amt': [100,200,150,300],
    'pay_dt': pd.to_datetime(['2024-01-02','2024-01-03','2024-02-02','2024-02-05'])
})

# 1
df = pd.merge(user, order, on='user_id', how='left')

# 2
user_agg = df.groupby('user_id').agg(
    order_cnt=('order_id','count'),
    total_amt=('pay_amt','sum'),
    first_pay=('pay_dt','min')
).reset_index()

user = user.merge(user_agg, on='user_id', how='left')

# 3
cond = (
    (user['register_dt'] >= '2024-01-01') &
    (user['channel'] == 'ad') &
    (user['order_cnt'] > 0)
)
result = user[cond]

# 4
user['is_pay'] = user['order_cnt'] > 0
conv_rate = user.groupby('channel')['is_pay'].mean()
print(conv_rate)
```

要求：

```python
import pandas as pd
from sklearn.preprocessing import OneHotEncoder, StandardScaler
from sklearn.feature_selection import VarianceThreshold
from sklearn.model_selection import train_test_split

df = pd.DataFrame({
    'age': [22,31,28,35,40],
    'salary': [5000,9000,7500,12000,14000],
    'gender': ['m','f','m','f','m'],
    'city': ['bj','sh','bj','sz','sh'],
    'churn': [0,1,0,1,0]
})

X = df.drop('churn', axis=1)
y = df['churn']

cat_cols = ['gender','city']
num_cols = ['age','salary']

# 编码
ohe = OneHotEncoder(drop='first', sparse_output=False)
cat_feat = ohe.fit_transform(X[cat_cols])
cat_df = pd.DataFrame(cat_feat, columns=ohe.get_feature_names_out())

# 标准化
scaler = StandardScaler()
num_feat = scaler.fit_transform(X[num_cols])
num_df = pd.DataFrame(num_feat, columns=num_cols)

X_all = pd.concat([num_df, cat_df], axis=1)

# 方差过滤
sel = VarianceThreshold(0.01)
X_selected = sel.fit_transform(X_all)

# 划分
X_train,X_test,y_train,y_test = train_test_split(
    X_selected, y, test_size=0.2, random_state=42, stratify=y
```

# 六、综合场景口述题（1 题 × 14 分 = 14 分）

**题目**：

你们数据中台每日生成用户宽表，但下游经常投诉：

指标不准、重复用户多、金额为负、缺失率高。

请你用 Python 设计一套**自动化数据质量监控方案**，讲清流程 + 核心代码思路。

## 参考答案（面试满分版）

### 流程

1. **数据探查**：统计缺失率、重复率、异常值比例。
2. **规则校验**：
    
    - 主键唯一约束
    - 非空约束
    - 数值范围约束（金额 ≥0）
    - 格式校验（手机号、身份证）
    - 业务逻辑校验（支付时间 > 注册时间）
    
3. **异常捕获**：输出不合格数据明细。
4. **质量报告**：每日生成质量分数、趋势图。
5. **自动修复**：可自愈问题自动处理，不可自愈则告警拦截。

### 核心代码框架

```python
def quality_check(df):
    # 1. 重复检查
    dup = df.duplicated('user_id').mean()
    # 2. 非空检查
    null_rate = df['pay_amt'].isnull().mean()
    # 3. 金额异常
    amt_error = (df['pay_amt'] < 0).mean()
    # 4. 手机号格式
    phone_invalid = ~df['phone'].str.match(r'^1\d{10}$').mean()
    
    # 异常数据
    bad_data = df[
        df.duplicated('user_id', keep=False) |
        (df['pay_amt'] < 0) |
        ~df['phone'].str.match(r'^1\d{10}$')
    ]
    bad_data.to_csv('quality_error.csv', index=False)
    
    return {
        'duplicate_rate': dup,
        'null_rate': null_rate,
        'amount_error_rate': amt_error,
        'phone_invalid_rate': phone_invalid
    }

# 自动修复
df['pay_amt'] = df['pay_amt'].clip(lower=0)
df = df.drop_duplicates('user_id', keep='last')
```
