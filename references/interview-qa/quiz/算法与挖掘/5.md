定位：大厂终面 /leader 面高频，偏**深度理解、踩坑经验、工程落地、抗压与思考**

## 1. 压力题：你说你会特征工程，那为什么模型上线后效果掉得很厉害？怎么定位、怎么救？

### **参考答案**

效果掉点一般就三类：**数据漂移、特征失效、样本分布不一致**。

1）先看**PSI / CSI**：特征分布是否偏移，尤其核心行为、时间类特征。

2）看**label 正确率**：线上真实 label 与训练 label 是否不一致（比如延迟反馈、定义变更）。

3）看**特征 pipeline**：离线统计窗口、缺失值填充、归一化方式是否与线上一致。

4）急救方案：

- 短期：关闭不稳定特征，回滚到上一版模型；
- 中期：重训模型，加入时序滑窗特征，增强鲁棒性；
- 长期：建立特征监控、自动重训、badcase 回流机制。

## 2. 原理深挖：XGBoost/LightGBM 为什么能在工业界几乎垄断表格数据？它的局限性在哪？什么场景不如深度学习？

### **参考答案**

优势：

- 对**噪声、缺失、异常值**鲁棒，不需要复杂预处理；
- 自带**特征交互、非线性拟合**，不用手动构造；
- 训练快、可解释性强（特征重要性、树路径），易上线易监控；
- 对小数据、高维稀疏数据依然稳定。

局限性：

- 无法捕捉**序列依赖、长程依赖**（用户行为序列、时间序列复杂模式）；
- 对高维稠密向量（Embedding）不友好，不如深度网络；
- 难以端到端学习多模态信息（文本 + 图像 + 表格）。

不如 DL 的场景：

- 用户行为序列建模、推荐召回、NLP / 图像相关预测、超高维 Embedding 场景。

## 3. RAG 落地坑题：你做过的 RAG 最容易翻车的三个点是什么？怎么系统性解决？

### **参考答案**

1）**文档切分不合理**

- 问题：上下文切断、关键信息被截断；
- 解决：按章节 / 段落切分，滑动窗口，保留标题层级元数据。

2）**检索不准导致幻觉**

- 问题：语义漂移、关键词漏召回；
- 解决：BM25 + 向量混合召回 + 重排，强制引用来源。

3）**知识库更新不及时、索引不一致**

- 问题：新增 / 删除文档不同步，回答过时；
- 解决：增量索引 + 全量周期重建 + 版本管理 + 校验机制。

## 4. 算法对比题：PCA、TSNE、UMAP 都是降维，分别在什么场景下必须用某一个而不能用别的？

### **参考答案**

- **PCA**：
    
    必须用：需要**线性解释、去共线性、加速后续模型**，且追求速度、可复现。
    
    不适合：非线性结构、可视化。
    
- **TSNE**：
    
    必须用：**高维数据可视化**（聚类效果展示）。
    
    不适合：后续建模用（丢失结构、无逆变换、慢）。
    
- **UMAP**：
    
    必须用：**既想保留全局结构，又想聚类效果好，还要比 TSNE 快**。
    
    工程首选：Embedding 降维 → 聚类 / 检索前置。

## 5. 开放性题：如果让你从零搭建一套 “内部文档智能问答系统”，你前两周会做什么？如何证明它有用？

### **参考答案**

前两周动作：

1）梳理数据源：确定文档范围、格式、更新频率；

2）快速搭建 baseline：抽取 → 切分 → BM25 + 向量召回 → LLM 生成；

3）标注一批测试集：构造真实高频问题，评估召回率与回答准确率；

4）优化最痛环节：切分、prompt、重排。

证明价值：

- 指标：**HitRate@10、回答准确率、人工满意度、问题解决率**；
- 业务：减少重复咨询、降低新人上手时间、提升查询效率。
