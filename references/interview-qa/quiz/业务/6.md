## 一、基础综合查询（30分）

### 标准答案

## 二、窗口函数综合应用（35分）

### 题目描述

基于第一题的三张表，新增需求：

1\. 计算各车间每日的生产总数、合格数、合格率（保留4位小数）；2\. 计算每个车间每日合格率较前一日的变化率（保留4位小数，变化率=（当日合格率\-昨日合格率）/昨日合格率，若昨日无数据则显示NULL）；3\. 计算每个车间当月（2024年10月）的平均合格率（保留4位小数）；4\. 标记出每个车间当日合格率是否低于当月平均合格率（低于显示“是”，否则显示“否”）；5\. 输出：车间名称、生产日期、当日生产总数、当日合格率、昨日合格率、变化率、当月平均合格率、是否低于月平均。

### 标准答案

```sql
WITH daily_workshop_prod AS (
    -- 第一步：计算各车间每日核心指标
    SELECT
        dw.workshop_name,
        dpdt.production_date,
        SUM(dpdt.total_num) AS daily_total,
        ROUND(SUM(dpdt.good_num) / SUM(dpdt.total_num), 4) AS daily_pass_rate
    FROM dwd_production_detail dpdt
    LEFT JOIN dim_workshop dw ON dpdt.workshop_id = dw.workshop_id
    WHERE dpdt.production_date BETWEEN '2024-10-01' AND '2024-10-31'
    GROUP BY dw.workshop_name, dpdt.production_date
),
workshop_month_avg AS (
    -- 第二步：计算各车间当月平均合格率
    SELECT
        workshop_name,
        ROUND(AVG(daily_pass_rate), 4) AS month_avg_pass_rate
    FROM daily_workshop_prod
    GROUP BY workshop_name
)
-- 第三步：关联窗口函数，计算昨日数据、变化率，标记是否低于月平均
SELECT
    dwp.workshop_name,
    dwp.production_date,
    dwp.daily_total,
    dwp.daily_pass_rate,
    LAG(dwp.daily_pass_rate, 1) OVER (PARTITION BY dwp.workshop_name ORDER BY dwp.production_date) AS yesterday_pass_rate,
    ROUND(
        (dwp.daily_pass_rate - LAG(dwp.daily_pass_rate, 1) OVER (PARTITION BY dwp.workshop_name ORDER BY dwp.production_date)) 
        / LAG(dwp.daily_pass_rate, 1) OVER (PARTITION BY dwp.workshop_name ORDER BY dwp.production_date),
        4
    ) AS pass_rate_change,
    wma.month_avg_pass_rate,
    CASE WHEN dwp.daily_pass_rate < wma.month_avg_pass_rate THEN '是' ELSE '否' END AS is_below_month_avg
FROM daily_workshop_prod dwp
LEFT JOIN workshop_month_avg wma ON dwp.workshop_name = wma.workshop_name
ORDER BY dwp.workshop_name ASC, dwp.production_date ASC;
```

### 思路解析

1\. 用CTE分层处理：先通过daily\_workshop\_prod计算每日车间指标，再通过workshop\_month\_avg计算月平均，逻辑更清晰，避免嵌套过多；2\. 窗口函数应用：LAG函数获取前一日合格率（按车间分区、日期排序），实现变化率计算；3\. CASE WHEN判断是否低于月平均，贴合业务分析需求；4\. 注意边界处理：昨日无数据时，LAG返回NULL，变化率也为NULL，符合实际业务逻辑。

## 三、复杂业务统计（35分）

### 题目描述

基于前两题的表结构，新增订单表（dwd\_order），表结构如下：

dwd\_order（订单明细）：order\_id（订单ID）、product\_id（产品ID）、order\_date（下单日期，格式：yyyy\-MM\-dd）、order\_num（订单数量）、actual\_deliver\_num（实际发货数量）、order\_amount（订单金额，单位：元）、status（订单状态：1\-待发货，2\-已发货，3\-已完成，4\-已取消）

需求（综合生产、产品、订单数据）：

1\. 统计2024年10月各产品的生产总量、订单总量（仅统计状态为2、3的有效订单）、发货总量、库存缺口（库存缺口=订单总量\-生产总量，若为负则显示0）；2\. 计算各产品的订单履约率（履约率=发货总量/订单总量，保留4位小数，仅统计有效订单且订单总量\&gt;0的产品）；3\. 按产品类型分组，计算每组的平均履约率、总订单金额；4\. 输出：产品类型、产品名称、生产总量、订单总量、发货总量、库存缺口、履约率、所属类型平均履约率、所属类型总订单金额；5\. 过滤掉产品类型为“耗材”的记录，按所属类型平均履约率降序、产品履约率降序排序。

### 标准答案

```sql
WITH product_prod AS (
    -- 1. 统计各产品10月生产总量
    SELECT
        dp.product_id,
        dp.product_name,
        dp.product_type,
        SUM(dpdt.total_num) AS production_total
    FROM dwd_production_detail dpdt
    LEFT JOIN dim_product dp ON dpdt.product_id = dp.product_id
    WHERE dpdt.production_date BETWEEN '2024-10-01' AND '2024-10-31'
    GROUP BY dp.product_id, dp.product_name, dp.product_type
),
product_order AS (
    -- 2. 统计各产品10月有效订单（状态2、3）的订单总量、发货总量、订单金额
    SELECT
        product_id,
        SUM(order_num) AS order_total,
        SUM(actual_deliver_num) AS deliver_total,
        SUM(order_amount) AS order_amount_total
    FROM dwd_order
    WHERE order_date BETWEEN '2024-10-01' AND '2024-10-31'
      AND status IN (2, 3)
    GROUP BY product_id
),
type_stats AS (
    -- 3. 按产品类型统计平均履约率、总订单金额
    SELECT
        pp.product_type,
        ROUND(AVG(CASE WHEN po.order_total > 0 THEN (po.deliver_total / po.order_total) ELSE NULL END), 4) AS type_avg_fulfillment,
        SUM(po.order_amount_total) AS type_total_amount
    FROM product_prod pp
    LEFT JOIN product_order po ON pp.product_id = po.product_id
    GROUP BY pp.product_type
)
-- 4. 关联所有表，计算最终指标并过滤排序
SELECT
    pp.product_type,
    pp.product_name,
    pp.production_total,
    COALESCE(po.order_total, 0) AS order_total,
    COALESCE(po.deliver_total, 0) AS deliver_total,
    -- 库存缺口：订单总量-生产总量，负则取0
    GREATEST(COALESCE(po.order_total, 0) - pp.production_total, 0) AS inventory_gap,
    -- 履约率：仅订单总量>0时计算，否则为NULL
    CASE WHEN COALESCE(po.order_total, 0) > 0 
         THEN ROUND(po.deliver_total / po.order_total, 4) 
         ELSE NULL END AS fulfillment_rate,
    ts.type_avg_fulfillment,
    ts.type_total_amount
FROM product_prod pp
LEFT JOIN product_order po ON pp.product_id = po.product_id
LEFT JOIN type_stats ts ON pp.product_type = ts.product_type
WHERE pp.product_type != '耗材'
ORDER BY ts.type_avg_fulfillment DESC, fulfillment_rate DESC;
```

### 思路解析

1\. 分层CTE：按“生产统计→订单统计→类型统计”分步处理，降低复杂查询难度，便于调试；2\. 空值处理：用COALESCE将NULL转为0，避免计算错误；3\. 业务逻辑处理：库存缺口用GREATEST函数确保非负，履约率用CASE WHEN过滤订单总量为0的情况，贴合实际业务；4\. 多表关联：关联生产、订单、类型统计结果，获取完整字段；5\. 过滤与排序：排除“耗材”类型，按要求排序，满足业务分析场景。
