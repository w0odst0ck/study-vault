- **背景**：生产、库存、订单数据分散在 ERP、MES、Excel 等多源系统，跨部门对账困难，报表依赖人工，决策滞后。

## 第 2 题

**问题**：在数仓开发中，**缓慢变化维（SCD）** 是非常常见的考点。请你解释 SCD1、SCD2、SCD3 的区别，并结合你的工厂项目说明你是如何处理维度变化的。

### **参考答案**

1. **SCD1 直接覆盖**
    
    新数据直接覆盖旧数据，不保留历史，适合不需要追溯的简单维度。
    
2. **SCD2 拉链表（最常用）**
    
    新增一行记录，通过 start_date、end_date、is_current 标记生命周期，完整保留历史版本。
    
3. **SCD3 加字段保留前后状态**
    
    增加如 previous_name 字段，只保留最近一次变化，不保留全历史。
    

**项目落地**

在工厂项目中，**产品信息、客户信息、车间工艺**等会不定期调整，我采用 **SCD2 拉链表**处理：

- 通过时间戳识别变化数据
- 新增生效记录，将旧记录置为失效
- 保证 DWD 明细与 DWS 汇总能按历史口径正确统计
    
    既满足了历史数据分析需求，又避免了全量覆盖导致的数据失真。

## 第 3 题

**问题**：请手写一段**复杂业务 SQL**，实现：按天统计每个车间的产量、合格率、不良数，并筛选出合格率低于 95% 的记录，同时展示同比昨日变化率。

（可使用窗口函数、CTE 等）

### **参考答案**

```sql
WITH daily_product AS (
    SELECT
        workshop_id,
        dt,
        SUM(total_num) AS total_num,
        SUM(bad_num) AS bad_num,
        ROUND(SUM(good_num)/SUM(total_num),4) AS pass_rate
    FROM dwd_production_detail
    GROUP BY workshop_id, dt
),
prod_yesterday AS (
    SELECT
        *,
        LAG(pass_rate,1) OVER (PARTITION BY workshop_id ORDER BY dt) AS pass_rate_yes,
        LAG(total_num,1) OVER (PARTITION BY workshop_id ORDER BY dt) AS total_num_yes
    FROM daily_product
)
SELECT
    workshop_id,
    dt,
    total_num,
    bad_num,
    pass_rate,
    ROUND((pass_rate - pass_rate_yes)/pass_rate_yes,4) AS pass_rate_change_ratio
FROM prod_yesterday
WHERE pass_rate < 0.95
ORDER BY dt, workshop_id;
```

思路：先用 CTE 做日聚合，再用 LAG 窗口函数取昨日数据，最后筛选低合格率并计算变化率。

## 第 4 题

**问题**：你在项目中用到了 **RAG + Milvus**，面试官可能会质疑：“向量检索也会丢信息、检索不准”。你如何保证在法律 / 金融这类严谨场景下的检索可靠性？请说出你的完整优化方案。

### **参考答案**

为保证法律、金融场景的检索严谨性，我从**召回、排序、过滤、校验**四层做保障：

1. **多路召回提高覆盖率**
    
    BM25 关键词召回 + Milvus 向量召回 + 元数据规则召回，避免单一方式漏召回关键法条。
    
2. **重排序提升精准度**
    
    对召回结果做交叉相似度排序，结合专业术语权重，优先保留高相关内容。
    
3. **强规则过滤**
    
    按法规生效日期、适用范围、条款等级过滤，剔除过期、无关内容。
    
4. **阈值截断与去重**
    
    设定相似度阈值，低于阈值不返回；对相似内容去重，避免重复干扰。
    
5. **结果校验机制**
    
    关键问答增加关键词命中校验，确保答案包含必要专业术语，提升可靠性。
    

最终实现高召回、高精准，满足法律金融场景的严谨性要求。

## 第 5 题

**问题**：如果让你**主导一个新公司的数据治理项目**，但领导不重视、业务不配合、资源也不足，你会怎么推进并做出成果？

### **参考答案**

我会采用**小切口、快落地、强价值**的思路分步推进：

1. **先抓最痛单点，不搞大而全**
    
    优先解决高频投诉点，如 “报表对不上”“取数慢”，快速见效。
    
2. **用数据成果说服领导**
    
    1～2 周内做出一个核心指标看板，展示效率提升、成本节约，争取资源与话语权。
    
3. **拉业务关键人共建**
    
    找各部门数据使用核心人员，统一指标口径，让业务参与并受益，降低推进阻力。
    
4. **建立最小可行治理规范**
    
    先出台表命名、字段口径、指标定义等极简规范，逐步完善，不一开始就强推复杂体系。
    
5. **自动化降低协作成本**
    
    通过 DataWorks 质量监控、自动化报表、告警机制，减少人工沟通成本。
    

最终以**可见价值 + 低侵入 + 强工具**逐步扩大范围，实现从单点优化到全面数据治理的落地。
