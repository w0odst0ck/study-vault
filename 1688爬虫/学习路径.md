### 📅 第 1 天：打破“浏览器黑盒” & 攻克第一道坎（编码与 DOM）

**目标**：搞清楚油猴脚本为什么能抓到数据，以及你的 `02_parse.py` 为什么能解析出 17 个字段。

- **上午（2h）：浏览器底层渲染原理（针对爬虫）**
  - 重点学：**DOM 树（Document Object Model）** 与 **CSS 选择器**。
  - 实战操作：打开 1688 搜索页，按 `F12` 打开开发者工具，用 `Elements` 面板手动追溯 `title`、`price`、`shop_name` 在 HTML 源码中的位置。
  - 搞懂 XPath 基础：在 Console 面板输入 `$x('//div[@class="title"]/a/text()')` 练习定位。
- **下午（4h）：解码你的 `02_parse.py`**
  - **重点攻克编码**：打开你的 `02_parse.py`，找到 `response.content.decode('gbk')` 这行。手动把这行改成 `utf-8` 看报什么错，彻底搞懂 `GBK` 和 `UTF-8` 在 bytes 层面的转换逻辑（这是 1688 新手最容易踩的坑）。
  - **拆解 BeautifulSoup**：把你代码里的 `soup.find_all` 复制出来，在 Jupyter Notebook 里逐行运行，打印出每个节点的 `text` 和 `attrs`，亲眼看看“年销量”和“回头率”藏在哪个 `data-xxx` 属性里。
- **晚上（2h）：油猴脚本原理入门**
  - 不用学 JS 语法，只需理解 **`window.scrollTo`** 如何触发浏览器的 `scroll` 事件，从而加载 Ajax 新数据。
  - 任务：打开 `01_fetch.user.js`，找到 `setInterval`，把翻页间隔从 3 秒改成 0.5 秒，运行后观察 1688 是否弹出滑块验证——**感受“频率控制”在反爬中的核心地位**。

### 📅 第 2 天：动态渲染对抗（Selenium） & 破解第三方解析器（急云/1688）

**目标**：搞懂 Selenium 到底在模拟什么，以及那个神奇的 `AlibabaParser` 是如何硬核提取 SKU 的。

- **上午（3h）：Selenium 底层原理（CDP 协议）**
  - 重点不是学 `find_element`，而是学 **Chrome DevTools Protocol (CDP)**。
  - 实战：打开 `06_detail_collector.py`，找到 `webdriver.Chrome` 的初始化参数。手动把 `options.add_argument("--headless")` 注释掉，让浏览器窗口弹出来，亲眼看着 Selenium 一页页加载详情页。
  - 任务：在 Selenium 启动的浏览器里，右键检查详情页的“品牌”字段，对比你的 `alibaba_parser.py` 里写的 XPath 是否精准匹配。
- **下午（3h）：攻克 `AlibabaParser` 的“硬编码逻辑”**
  - 打开 `1688/utils/parsers/alibaba_parser.py`，你会看到大量的 `self.xpath('//div[@class="sku-wrap"]')`。
  - **核心任务**：找一个你之前采集失败的详情页 HTML，用记事本打开，手动搜索 `brand` 或 `货号`，然后在 `alibaba_parser.py` 里新增一条自己的 XPath 提取规则（比如提取“材质”）。
  - 搞懂 `get_sku_matrix()`：它是如何把“颜色:红色|尺寸:L”这种字符串切分成字典的（涉及 Python 的 `split` 和 `zip` 联动）。
- **晚上（2h）：资源下载机制（subprocess）**
  - `07_process_with_1688lib.py` 调用了 aria2c。重点学习 Python 的 `subprocess.Popen`，搞懂它是如何把主图 URL 列表通过管道（Pipe）喂给外部 exe 程序的。

### 📅 第 3 天：管线工程化（Argparse + Pathlib）& 全链路联调

**目标**：不再局限于单文件，站在架构师视角看 `run1.py` 和 `run2.py` 的数据流动。

- **上午（3h）：工程化命令行与文件系统**
  - 重点研究 `argparse` 模块：看 `run1.py` 如何通过 `--cat 投光灯` 把参数传给 `02_parse.py`。
  - 搞懂 `pathlib.Path`：手动 `print` 出 `data/投光灯/` 下的所有文件路径，理解 `mkdir(parents=True, exist_ok=True)` 是如何防止报错的（这在批量创建品类文件夹时极重要）。
- **下午（4h）：** **手动“脏调试”全链路**
  - 关闭自动运行的脚本，**手动分步执行**：

    2. 跑完后，手动打开生成的 `all_products.csv`，用 Pandas 看一眼数据形状。
    3. 跑 `03_clean.py`，重点看它是用 `drop_duplicates` 还是 `groupby` 去重的（结合你的 Numpy 基础理解底层向量化）。
    4. 跑 `04_analyze.py`，**挑战**：把 Matplotlib 的默认颜色风格换成 `ggplot` 或 `seaborn-dark`，让 `analysis_chart.png` 更好看。
- **晚上（2h）：总结“反爬特征”**
  - 打开浏览器开发者工具的 `Network`（网络）面板，刷新 1688 搜索页，看看返回的 HTML 里是否包含 `sign`、`token` 等加密参数。
  - **结论输出**：用自己的话总结——为什么你的项目选择“油猴+Selenium（模拟真人）”而不是“requests 直接调用 API”？把这个逻辑写进你的项目 Wiki 里，作为技术摸底的收官之作。
