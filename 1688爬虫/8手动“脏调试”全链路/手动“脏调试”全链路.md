### 2.2 结合 NumPy 理解底层数据类型（为什么 Pandas 快？）
在检查输出时，注意 `df.info()` 的 `Dtype` 列。

- 如果 `price` 是 `object`（字符串），说明数字没转过来，后续无法做均价计算。
- **底层原理**：Pandas 的 `object` 列本质上是一个 **Python 指针数组**，遍历极慢。而 `float64` / `int64` 是 **C 语言级别的连续内存块（NumPy 数组）**，向量化运算速度相差 **100 倍以上**。

**手动修复提示**：如果在预览中发现价格带 `¥` 符号，记下来——你需要在 `03_clean.py` 里看它是否用 `.str.replace()` 处理了。这是下午检查 `03_clean.py` 代码时的关键观察点。

## Part 3：去重深度剖析 —— `drop_duplicates` vs `groupby`（75 分钟）

打开 `pipeline/03_clean.py`，找到清洗的核心逻辑。我们重点关注它是**如何干掉重复商品的**。

### 3.1 定位去重代码（两种常见写法）

**写法 A：`drop_duplicates`（你大概率会看到这个）**
```python
# 假设重复的定义是：同一个店铺卖同一个标题
df_cleaned = df.drop_duplicates(subset=['title', 'shop_name'], keep='first')
```
- **底层原理**：Pandas 调用 NumPy 的 `unique` 或哈希表（Hash Table）算法，对指定列的组合进行**向量化哈希计算**。整个过程在 C 语言级别完成，**完全不涉及 Python 级别的 for 循环**。
- **适用场景**：简单的“完全重复行”剔除。

**写法 B：`groupby` + `agg`（聚合去重）**
```python
# 如果同一个商品有多个价格，取最低价
df_cleaned = df.groupby(['title', 'shop_name'], as_index=False).agg({
    'price': 'min',        # 取最低价
    'yearly_sales': 'first' # 保留第一个销量
})
```
- **底层原理**：`groupby` 涉及**分组-应用-组合（Split-Apply-Combine）**。虽然也有向量化优化，但多了一步聚合函数计算，比 `drop_duplicates` 稍慢一些，但功能更强大。

### 3.2 你的“脏调试”任务（对比实验）
在 Jupyter 里用你的 `all_products.csv` 手动复现一遍，感受速度差异：

```python
import time

# 计时：drop_duplicates
start = time.time()
df_dedup = df.drop_duplicates(subset=['title', 'shop_name'])
print(f"drop_duplicates 耗时: {time.time() - start:.4f} 秒")

# 计时：groupby 同样功能（取第一行）
start = time.time()
df_group = df.groupby(['title', 'shop_name'], as_index=False).first()
print(f"groupby 耗时: {time.time() - start:.4f} 秒")

# 检查两个结果的行数是否一致
print(f"去重后行数: {len(df_dedup)} vs {len(df_group)}")
```
**预期结论**：`drop_duplicates` 通常快 20%~30%。结合你的 NumPy 基础，理解这是因为 `drop_duplicates` 直接在内存连续块上操作，而 `groupby` 需要额外的分组索引开销。

**观察代码**：看 `03_clean.py` 用了哪种策略，并思考——如果数据量扩大到 10 万行，当前策略是否依然高效？

**可能会翻车的地方（中文字体变方框）**：
由于 `ggplot` 主题会强制修改字体，如果你的系统没有中文字体，图例里的“投光灯”、“销量”会变成小方框 `□□`。

**急救方案（在 `plt.style.use` 后强制指定中文字体）**：

```python
import matplotlib.pyplot as plt

plt.style.use('ggplot')

# 修复中文字体（Windows 用 SimHei，Mac 用 Arial Unicode MS）
plt.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei', 'Arial Unicode MS']
plt.rcParams['axes.unicode_minus'] = False  # 解决负号显示问题
```

## 🚨 翻车预警 & 急救包

| 现象 | 排查方法 |
| :--- | :--- |
| `KeyError: 'title'` | `02_parse.py` 解析失败，没生成 `title` 列。检查 `all_products.csv` 是否为空，或在 Jupyter 里 `df.columns` 看字段名是否变了（比如 `Title` 大小写敏感）。 |
| Matplotlib 报错 `TypeError: no numeric data to plot` | `03_clean.py` 没把 `price` 转成数字。检查清洗代码里是否有 `pd.to_numeric(errors='coerce')`。 |
| 使用 `ggplot` 后图例消失了 | `ggplot` 的默认图例位置可能被移出画布。在 `plt.legend()` 前加 `plt.tight_layout()` 重新调整布局。 |
| 更改主题后图表变成“大黑块” | 可能是深色主题（如 `seaborn-dark`）配合白色坐标轴刻度看不见。加一行 `plt.rcParams['axes.labelcolor'] = 'white'`。 |
