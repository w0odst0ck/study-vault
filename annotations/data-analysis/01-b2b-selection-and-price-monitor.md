# 注：B2B 选品分析与缺口匹配

## L1 术语

### 移动平均 [ⓘ]
一种平滑数据的方法，用过去 N 个时间点的平均值作为当前基准。
价格监控中，用移动平均 vs 当前价做对比，比逐次比较更稳定（避免短期波动误报）。

### 冷启动 [ⓘ]
新商品上线初期，数据样本少（快照 < 3 次），此时计算的价格趋势不可靠。
方案：不给足够的样本数不做预警，避免冷启动阶段的误报。

## L2 概念

### 四层选品模型 [ⓘ]
| 层级 | 判定逻辑 | 动作 |
|------|---------|------|
| 🟢必上 | 强需求 + 高利润 | 立即上架 |
| 🔵推荐 | 有潜力（需求/利润之一强） | 优先上架 |
| 🟡暗马 | 长尾 + 竞品覆盖少 | 试探性上架少量 |
| 🔘关注 | 趋势待观察 | 先不上架，继续监控 |

本质是**风险收益权衡**：确定性高的重投入，不确定性高的轻投入。

## L3 架构

### Streamlit → Flask 的技术路线转变 [ⓘ]
Streamlit：快速原型工具，适合数据分析展示。
Flask：全功能 Web 框架，适合做产品。
Streamlit 做原型验证 → Flask 做正式产品，这是典型的数据产品开发路径。
不是技术原因，而是产品化需求（权限/路由/扩展性）决定。

### 品牌提取的"双重解析"策略 [ⓘ]
品牌提取准确率从 0%→91%，关键在**融合两个弱信号**：
- 标题中的品牌名（可能有，可能被截断）
- 店铺名中的品牌（可能有，与标题互补）
两者交叉验证，比单一来源可靠得多。
