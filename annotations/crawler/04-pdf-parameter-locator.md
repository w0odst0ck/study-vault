# 注：PDF 参数页定位器实战

## L1 术语

### PyMuPDF (fitz) [ⓘ]
Python 的 PDF 处理库，底层用 C 写的 MuPDF 引擎。
提取文字性能极好：59 页 PDF 只需 0.12 秒。
比 pdfplumber 快但提取结果更原始（没有表格结构）。

### pymupdf4llm [ⓘ]
基于 PyMuPDF，专门把 PDF 转 Markdown 格式。
含表格识别（Pipe Table），但单元格内容有碎片（同一个格子的文字可能被拆分）。
适合 LLM 读取，不适合精确参数提取。

### Tesseract [[glossary/Tesseract]] [ⓘ]
见 [glossary/Tesseract](../glossary/Tesseract.md)。

### EasyOCR [ⓘ]
纯 pip 安装的 OCR 库，底层带 PyTorch（~500MB）。
优点是开箱即用（不需要系统安装），缺点是模型大、速度慢。

## L2 概念

### 为什么定位优于提取 [ⓘ]
OCR 中文表格精度只有 60-70%，自动提取的数值不可靠。
但是定位参数在哪几页（59 页 → 5 页）是非常可靠的任务（规则匹配）。
**策略**：让机器做它擅长的事（定位），让人类做机器做不好的事（精确提取）。

### 规则分级 P0/P1/P2 [ⓘ]
三级规则的置信度不同：
- P0：确定性极高（"配光性能"就是参数页），命中 1 个就够
- P1：概率性高（"照度"可能出现在参数页也可能出现在参考页），要多命中几个
- P2：辅助验证（"最小值"本身不是参数特征），需要跟 P1 配合

## L3 架构

### 两条路线 + 统一规则 [ⓘ]
文字层和 OCR 路线共享同一套关键词规则。
好处：规则维护一份，两条路线各自用自己的方式检测。
如果路线专用规则，维护量翻倍且容易不一致。
