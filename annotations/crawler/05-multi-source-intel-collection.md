# 注：多数据源情报采集（Intel 模块）架构

## L1 术语

### RSS/Atom [ⓘ]
网站内容更新的标准订阅格式（XML 格式）。
类似微信公众号的订阅机制——你订阅一个 RSS 源，有新内容自动推送。
结构化、稳定、易解析，是爬虫最友好的数据源。

### MD5 去重 [ⓘ]
对内容计算 MD5 哈希值（32 位十六进制字符串），存到文件里。
下次采集时先算 MD5，跟历史记录比，相同就跳过。
优点是计算快、占用小；缺点是内容微调（加个空格）会生成不同 Hash。

## L2 概念

### 数据源协议类型 [ⓘ]
| 协议 | 稳定性 | 解析难度 | 建议 |
|------|--------|---------|------|
| 官方 API | 高 | 低 | 首选 |
| RSS/Atom | 中 | 低 | 次选 |
| HTML 解析 | 低 | 中 | 后备 |
| JS 渲染 | 低 | 高 | 避免 |

### 移除教训 [ⓘ]
数据源不是越多越好。移除的 7 个源各有原因：
- arXiv → 学术论文，不适用宏观科技资讯
- GlobeNewswire → RSS 400 限制
- 东方财富 → 418 反爬，JS 渲染
- 央行 → connection reset（最隐蔽的错误）

## L3 架构

### LLM 安全策略 [ⓘ]
LLM 默认禁用 + `--llm` 按需启用 + 失败返回 None。
这种设计体现了**功能降级**思想：核心功能不依赖外部服务。
即使 LLM 挂了，规则分类仍然能保底产出。
