# 独立站竞品逛站分析方法论

> 系统化逛竞品站，避免「看一眼就过」的无效调研

## 逛站前的准备工作

1. **建好截图目录** — 按站点分文件夹
2. **准备 checklist** — 统一衡量维度，跨站对比
3. **分段逛站** — 每次集中攻一个站，不要同时在多个站之间跳转
4. **随手笔记** — 设计灵感点立刻记，不信任大脑记忆

## Checklist 维度

### 视觉层
- 品牌色系（主色 + 辅助色 + 色值提取）
- 字体使用（英文字体 / Arabic 字体）
- Banner 视觉风格（实拍图 vs 合成 vs 3D）
- 产品图风格（白底、场景、模特）
- 整体留白 / 间距 / 呼吸感

### 内容层
- 首页模块结构（banner → 分类 → 热销 → 关于）
- 产品页结构（标题 → 参数 → 描述 → 评价 → 推荐）
- 导航结构（顶部 / 侧边 / 汉堡菜单）
- Footer 内容（联系方式 / 政策链接 / 社交媒体）

### 体验层
- 语言切换（单一语种 vs 多语言）
- 货币 / 地区切换
- 搜索功能
- 加载速度
- 移动端适配

### 技术层
- 建站工具检测（BuiltWith / Wappalyzer）
- CDN / SSL
- 支付方式接入
- 评论/评分系统
- 多语言插件

## 分段逛站路线

对 6+ 个竞品站，按以下路线安排：

```
第1站 → 观察首页和整体印象（不深究细节）
第2站 → 对比首页细节 + 截图
第3站 → 产品页对比
第4站 → 结算流程体验
第5站 → 技术栈检测 + 性能评估
第6站 → 追溯（补缺漏 + 最佳实践汇总）
```

每站结束后写简短总结（3-5 句），不要「这个站好看」这种模糊描述，要写具体「哪好看 + 为什么好看 + 适不适合自己」。

## 关注的重点

- **不要抄** — 看同行是为了知道自己不要做什么 + 找到差异化方向
- **找模式** — 哪些模块结构反复出现（行业共识），哪些是独特点
- **做减法** — 竞品有的不一定要有，判断是否真正需要
- **截图即证据** — 所有判断附截图，方便后续复盘核对

## 回顾

- Q: 逛竞品站的 checklist 分哪几个维度？
  A: 视觉层（色彩/字体/风格）、内容层（模块结构）、体验层（语言/速度/适配）、技术层（建站工具/支付/CDN）。
- Q: 分段逛站路线怎么安排？
  A: 6站分阶段：首页印象 → 首页细节 → 产品页 → 结算流程 → 技术分析 → 趋势补漏。不建议同时跳多个站。
- Q: 逛站笔记的最低质量要求是什么？
  A: 写具体对比：不说「这站好看」，说「哪好看 + 为什么好看 + 是否适合自己」。每人站配截图 + 3-5句总结。
- Q: 逛竞品站的核心原则？
  A: 不要抄，找差异化和模式共识；做减法，判断是否真的需要；截图即证据。
- Q: 逛站总结表包含哪些栏目？
  A: 品牌色、首页布局、产品页、亮点、避坑，最后一列留自己的方向决策。
