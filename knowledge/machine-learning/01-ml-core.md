## 1. 模型对比速查

| 模型 | 适用 | 优点 | 缺点 |
|------|------|------|------|
| 逻辑回归 | 二分类 | 可解释强、概率输出 | 线性决策边界 |
| 决策树 | 分类/回归 | 直观、无需特征缩放 | 易过拟合 |
| 随机森林 | 分类/回归 | 抗过拟合、特征重要性 | 模型大、慢 |
| XGBoost | 分类/回归/排序 | 精度高、正则化 | 调参复杂 |
| LightGBM | 大数据 | 比 XGBoost 快、省内存 | 小数据易过拟合 |
| K-Means | 无监督聚类 | 简单高效 | 需指定 K、对初始值敏感 |
| PCA | 降维/可视化 | 去噪、减少特征数 | 可解释性丢失 |

### Boosting vs Bagging

| | Bagging（RF） | Boosting（XGB/LGB） |
|--|--------------|-------------------|
| 训练方式 | 并行 | 串行 |
| 重点 | 降低方差 | 降低偏差 |
| 抗过拟合 | 强 | 弱（需正则化） |

## 2. 评估指标

### 分类

```
准确率 = (TP+TN) / (TP+TN+FP+FN)    # 数据平衡时用
精确率 = TP / (TP+FP)                 # 关注"预测为真"的准确度
召回率 = TP / (TP+FN)                 # 关注"找全所有真值"
F1 = 2 * P*R / (P+R)                 # 精确率和召回率的调和平均
AUC-ROC                              # 模型排序能力，不依赖阈值
```

### 回归

```
MSE = mean((y_pred - y_true)²)
MAE = mean(|y_pred - y_true|)
R² = 1 - SS_res / SS_tot
```

## 3. 特征工程

| 步骤 | 方法 | 面试常问 |
|------|------|---------|
| 缺失值处理 | 均值/中位数/众数填充、模型预测 | 为什么用均值？类别特征用众数 |
| 异常值处理 | IQR、Z-Score、DBSCAN | 异常值是否要删？看业务场景 |

| 特征选择 | 过滤式/包裹式/嵌入式 | L1 正则化天然特征选择 |

## 4. 过拟合与欠拟合

| | 过拟合 | 欠拟合 |
|--|--------|--------|
| 训练误差 | 很低 | 很高 |
| 验证误差 | 很高 | 很高 |
| 原因 | 模型太复杂、数据太少 | 模型太简单、特征不足 |
| 解决 | 正则化、更多数据、早停、剪枝 | 更复杂模型、更多特征、减少正则化 |

### 防止过拟合的常用方法

```python
# L1/L2 正则化 — 在损失函数加惩罚项
# Dropout — 训练时随机丢弃神经元
# 早停 — 验证集不改善时停止训练
# 数据增强 — 增加训练样本多样性
# 交叉验证 — 评估泛化能力
```

## 回顾

- Q: 随机森林为什么能抗过拟合？
  A: Bagging 多棵树平均降低方差 + 随机选择特征增加多样性

- Q: XGBoost 相比 GBDT 的改进？
  A: 二阶泰勒展开（比一阶更准）、正则化项、列抽样、并行近似直方图

- Q: 特征缩放为什么对线性模型必要，对树模型不必要？
  A: 线性模型梯度更新受特征尺度影响，树模型按特征值切分不受尺度影响

- Q: AUC 为 0.5 和 1.0 分别代表什么？
  A: 0.5 完全随机，1.0 完美区分。AUC 衡量模型排序能力，与阈值无关

- Q: 类别不平衡怎么处理？
  A: 过采样（SMOTE）、欠采样、调整类别权重、使用 AUC 等不敏感指标
