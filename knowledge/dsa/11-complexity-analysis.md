# 复杂度分析

> 大 O / 大 Θ / 均摊分析 / Master Theorem
> 面试高频：考算法必问复杂度

---

## 1. 大 O 速查表

### 常见量级

| 记号 | 名称 | 典型场景 | n=10⁶ 时的操作数 |
|------|------|---------|-----------------|
| O(1) | 常数 | 哈希查找、数组索引 | 1 |
| O(log n) | 对数 | 二分查找、平衡树 | ~20 |
| O(n) | 线性 | 数组遍历 | 10⁶ |
| O(n log n) | 线性对数 | 快排/归并/堆排 | ~20×10⁶ |
| O(n²) | 平方 | 冒泡/选择/暴力 DP | 10¹² ❌ |
| O(2ⁿ) | 指数 | 子集枚举、无剪枝回溯 | 不可行 |
| O(n!) | 阶乘 | 全排列暴力 | 不可行 |

### 速算技巧

```
一层循环          → O(n)
两层嵌套循环      → O(n²)
分治（T=2T(n/2)+n）→ O(n log n)
二分              → O(log n)
递归指数爆炸      → O(2ⁿ)
```

## 3. 均摊分析

有些操作偶尔代价高、但大部分时候低——用**均摊**看整体。

### 动态数组扩容

```python
# Python list / C++ vector 的 append
# 平时 O(1)，满时 O(n) 扩容
# 均摊：O(1)
```

**计算**：扩容倍增 → n 次插入总开销 2n → 均摊 O(1)

### 单调栈

每个元素入栈一次出栈一次 → 总 O(n) → 均摊 O(1)

## 4. Master Theorem（主定理）

用于**分治递归**的复杂度分析。

```
T(n) = a·T(n/b) + f(n)

a ≥ 1: 子问题数量
n/b:   每个子问题大小
f(n):  分解+合并的代价
```

### 三种情况

| 条件 | 复杂度 | 例子 |
|------|--------|------|
| f(n) = O(n^(log_b(a) - ε)) | T(n) = Θ(n^log_b(a)) | 二叉树遍历 T(n)=2T(n/2)+O(1) → O(n) |
| f(n) = Θ(n^log_b(a)) | T(n) = Θ(n^log_b(a) · log n) | 归并排序 T(n)=2T(n/2)+O(n) → O(n log n) |
| f(n) = Ω(n^(log_b(a) + ε)) | T(n) = Θ(f(n)) | — |

### 速算口诀

```
二分查找   T(n) = T(n/2) + O(1)  → O(log n)
二叉树遍历 T(n) = 2T(n/2) + O(1) → O(n)
归并排序   T(n) = 2T(n/2) + O(n) → O(n log n)
全二叉树   T(n) = 2T(n/2) + n    → O(n log n)  # 不常见
```

## 5. 面试常用分析套路

| 场景 | 分析方法 |
|------|---------|
| 单循环 | O(n) — 看循环次数 |
| 双循环（非嵌套） | O(n+m) — 两个独立循环 |
| 双循环（嵌套） | O(n²) — 除非内层有条件提前退出 |
| 排序+遍历 | O(n log n) — 排序是瓶颈 |
| 二分+遍历 | O(n log n) — 二分 O(log n) × 遍历 O(n) |
| 递归树高 h | O(h) — 树相关算法空间复杂度 |

## 回顾

- Q: O(1)、O(n)、O(n²)、O(n log n) 分别代表什么规模？
  A: O(1) 常数、O(n) 线性、O(n log n) 线性对数（排序级）、O(n²) 平方（暴力级）

- Q: 动态数组 append 的均摊复杂度为什么是 O(1)？
  A: 扩容倍增策略下，n 次插入总开销约 2n，均摊到每次为 O(1)

- Q: 归并排序用 Master Theorem 怎么算？
  A: T(n) = 2T(n/2) + O(n) → a=2, b=2, f(n)=n → log_2(2)=1 → 情况2 → Θ(n log n)

- Q: 为什么快排最坏 O(n²) 但实际很快？
  A: 最坏发生在已有序+每次选端点 pivot；随机 pivot / 三数取中后概率极低，平均 O(n log n)

- Q: 树相关算法的空间复杂度怎么看？
  A: 递归深度 = 树高 h，平衡树 h=log n，链状树 h=n。空间就是 O(h)
