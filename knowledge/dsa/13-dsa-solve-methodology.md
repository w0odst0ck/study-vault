## 1. 暴力→优化的思维路径

### 两数之和的思维转变

```
暴力：for i → for j → O(n²)
观察：target - nums[i] = 要找的值
优化：哈希表存已遍历的值 → O(n)
```

**模式识别**：当暴力解法涉及"在集合中查找"时 → 考虑哈希表

### 滑动窗口的思维转变

```
暴力：枚举所有子串 → O(n²)
观察：窗口向右滑动时，左边界也向右 + 窗口内容只增不减
优化：维护窗口内字符频率 → O(n)
```

**模式识别**：连续子数组/子串问题 → 滑动窗口比前缀和更易写

## 2. 边界条件心法

### 二分查找边界检查清单

每次写二分前默念：

| 检查项 | 正确做法 |
|--------|---------|
| 循环条件 | `left <= right` 还是 `left < right`？ |
| mid 计算 | `left + (right-left)//2` 防溢出 |
| 收缩方向 | 找到 target 后向左还是向右继续找？ |
| 返回值 | 索引还是插入位置？找不到返回 -1 还是 left？ |

**口诀**：先定循环条件，再定收缩方向，最后确认返回值

### 链表操作检查清单

| 检查项 | 正确做法 |
|--------|---------|
| 头节点可能为空？ | 用 `dummy` 节点 |
| 指针推进了吗？ | 每次操作后 `cur = cur.next` |
| 循环终止条件？ | `while cur and cur.next` |

## 3. 递归三问法

写递归前先问自己三个问题：

1. **终止条件是什么？**（什么时候 return）
2. **返回值是什么？**（返回给谁用）
3. **这一层做什么？**（当前节点/元素的处理）

### 举例：二叉树最大深度

```python
def max_depth(root):
    # Q1: 终止条件
    if not root:
        return 0
    # Q3: 这一层做什么？——取左右子树最大深度
    left = max_depth(root.left)
    right = max_depth(root.right)
    # Q2: 返回值是什么？——当前子树深度
    return max(left, right) + 1
```

**模式识别**：递归卡住时，画出调用栈的**展开**和**回溯**两个阶段

## 4. 错题复盘方法

### 正确的复盘步骤

```
1. 先自己写（哪怕暴力）→ 暴露真实问题
2. 看题解理解思路 → 不要直接抄代码
3. 关掉题解重新写 → 检验是否真懂
4. 记录卡点和转换思路 → 下次不再犯
```

### 错误的复盘方式

```
❌ 看一遍题解 → "懂了" → 下一题
❌ 抄了一遍代码 → 不重写 → 下次还是不会
❌ 只记答案不记思维过程 → 题型一变就懵
```

## 5. 时间分配策略

| 阶段 | 时间 | 动作 |
|------|------|------|
| 看题 | 3 min | 理解题意、想暴力解法 |
| 思考 | 5 min | 尝试优化思路 |
| 写代码 | 10-15 min | 独立写 |
| 调试 | 5 min | 自己找 bug |
| 看题解 | — | 卡住就看，别死磕 |
| 重写 | 10 min | 关掉题解独立重写 |

**关键**：不会就看题解，但必须**自己重写一遍**，否则等于没做。

## 回顾

- Q: 暴力→优化的第一步是什么？
  A: 观察暴力解法中哪一步最耗时（通常是在集合中查找），然后用哈希表缓存

- Q: 递归三问是哪三个问题？
  A: ①终止条件是什么？②返回值是什么？③这一层做什么？

- Q: 链表操作中 dangling pointer 怎么防？
  A: 用 dummy 节点保底，每次操作后推进指针（`cur = cur.next`），提前想好循环终止条件

- Q: 看题解后怎么确保真懂了？
  A: 关掉题解，独立重写一遍。写不出来说明没真懂，再回去看

- Q: 滑动窗口和前缀和怎么选？
  A: 需要找"恰好满足条件"的连续子数组 → 滑动窗口。需要频繁计算区间和 → 前缀和
