# 数据分析面试核心：AB实验、因果推断、指标体系

> 数据岗面试必考三件套
> 来源：旧知识库「Knowledge-Base-for-Interviews」

---

## 1. AB 实验

### 核心流程

```
提出假设 → 设计实验 → 分流 → 运行 → 分析结果 → 决策
```

### 关键概念

| 概念 | 说明 |
|------|------|
| 原假设 H₀ | 实验组和对照组无差异 |
| 备择假设 H₁ | 实验组和对照组有差异 |
| p 值 | 在原假设为真时，观察到当前结果的概率 |
| 显著性水平 α | 通常 0.05，p < α 则拒绝 H₀ |
| 统计功效 1-β | 正确拒绝 H₀ 的概率，通常要求 > 0.8 |
| MDE | 最小可检测效应 |

### 常见陷阱

```python
# ❌ 多重比较：同时对多个指标跑假设检验
# ✅ 用 Bonferroni 校正：α' = α / 比较次数

# ❌ 早停：看到 p<0.05 就提前结束
# ✅ 预先确定样本量和运行时长

# ❌ 分流不均匀：用户 ID hash 不均匀
# ✅ 用稳定的 hash 函数 + 分层分流
```

## 2. 因果推断

### 核心问题

**关联 ≠ 因果**。观察到的相关性可能来自混杂变量。

### 常用方法

| 方法 | 适用场景 | 核心思想 |
|------|---------|---------|
| 随机实验（RCT） | 条件允许时 | 随机分组消除混杂 |
| DID（双重差分） | 政策/干预前后 | 实验组-对照组在干预前后的差值之差 |
| 工具变量 IV | 存在不可观测的混杂 | 找一个只影响 Treatment 不影响 Outcome 的变量 |
| 断点回归 RDD | 阈值附近 | 利用阈值附近的随机性 |
| 倾向性得分匹配 PSM | 观察数据 | 根据倾向性得分匹配相似的个体 |

### 辛普森悖论

```python
# 分组看正相关，合并看负相关
# 原因：混杂变量（如样本量分布不均）
# 解决：按层分析后加权汇总
```

## 3. 指标体系

### 指标分级

| 层级 | 示例 | 作用 |
|------|------|------|
| 北极星指标 | 用户活跃度、GMV | 公司级唯一方向 |
| 一级指标 | 留存率、转化率 | 核心业务健康度 |
| 二级指标 | 访问深度、客单价 | 过程指标 |
| 三级指标 | 按钮点击率、页面加载时长 | 具体可执行指标 |

### 常见框架

- **OSM 模型**：Objective（业务目标）→ Strategy（策略）→ Measurement（度量）
- **AARRR 海盗模型**：Acquisition → Activation → Retention → Revenue → Referral
- **GSM 模型**：Goal → Signal → Metric

### 指标异动归因

```
指标下滑 → 拆维度（时间/地区/渠道/用户分层）→ 定位异常层 → 下钻分析 → 归因
```

## 回顾

- Q: AB 实验中的多重比较问题怎么解决？
  A: Bonferroni 校正（α除以比较次数）、FDR 控制、先定主指标

- Q: AB 实验什么时候可以提前停止？
  A: 原则上不能。如需提前停止，用序贯检验（sequential testing）而非固定样本检验

- Q: 辛普森悖论是什么？怎么避免？
  A: 分组正相关但总体负相关，由混杂变量导致。解决方法：按层分析 + 加权汇总

- Q: 因果推断的黄金标准是什么？现实替代方案是什么？
  A: 黄金标准是 RCT（随机对照实验）。现实用 DID、IV、RDD、PSM 等方法

- Q: 北极星指标和虚荣指标的区别？
  A: 北极星指标直接反映业务价值且可行动（如 DAU），虚荣指标看起来好看但无法指导行动（如累计注册用户数）
